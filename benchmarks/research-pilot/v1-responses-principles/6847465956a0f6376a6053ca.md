# Modified U-Net for Instance Segmentation of Noisy Fluorescence Microscopy

## Problem framing & domain constraints
**Task:** instance segmentation of a 2D single-plane, possibly multi-channel (nucleus/membrane/organelle markers — a feature dimension, NOT z-depth) fluorescence image. Output: per-pixel integer label map, one unique ID per cell/organelle, background=0. This is NOT foreground-vs-background: a binary mask collapses two touching objects into one connected component. Brightness is not a reliable identity cue (fluorophore expression, illumination, photobleaching all vary).

**Why touching objects are the core difficulty:** the inter-object gap is often a few dim pixels, easily erased by Poisson/read noise or illumination inhomogeneity. A foreground mask is structurally incapable of resolving this, so an explicit instance-separation signal beyond binary masking is mandatory.

**Constraints:** unique-label output; explicit separation at near-zero spacing; low-SNR (Poisson+read noise) robustness; intensity-inhomogeneity invariance (no brightness thresholds); preserve small objects/fine detail; dense packing; no natural-image assumptions; 2D single-plane (no z-context); variable channel count (not RGB); severe class imbalance; separate via geometry not absolute intensity.

## The design spine: chosen representation
The network predicts, ALONGSIDE a foreground channel, a **per-object gradient flow field (2 ch) + per-object max-normalized distance transform (1 ch)**. Candidates weighed: boundary/3-class (poor at low SNR — the separation cue is exactly the pixels noise destroys); distance transform (smooth, but peaks merge for tiny adjacent objects); gradient flow (STRONG — flows converge per-object, shape-agnostic, direction is intensity-free); embeddings (unstable clustering, must guess instance count). Flow is the strongest splitter; distance stabilizes/seeds sinks and guarantees tiny dim objects form a peak; both are pure geometry honoring inhomogeneity. Output tensors: Flow-Y/Flow-X (unit vectors, [-1,1], bg=0), normalized distance ([0,1], bg=0). Per-object normalization (never global); regression masked to foreground in the loss.

## (a) Architecture — Attention-Gated U-Net, triple heads
Symmetric 5-level (4-downsampling) encoder-decoder with 4 skip connections — an unmistakable U-Net. Single committed enhancement: additive attention gates on the skips (Oktay-style). Input: C channels (variable, not hardcoded 3), H×W divisible by 16, per-channel percentile normalization (clip 1st-99th pct → [0,1], no global scaling). Encoder: [3×3 conv→GroupNorm(8)→ReLU]×2, widths 32→64→128→256→512; stride-2 conv downsampling (not maxpool — avoids amplifying shot-noise peaks); GroupNorm because microscopy batches are tiny. Attention gate: α=σ(ψ·ReLU(W_x·x+W_g·g))∈[0,1], gated skip x̂=α⊙x. Bottleneck: 512-ch ConvBlock at H/16 (no transformer). Decoder: bilinear-upsample+conv (not transpose conv — checkerboard-free for smooth regression fields), concat gated skip, refine. Heads (4 channels total): semantic 32→1 Sigmoid; flow 32→2 Tanh; distance 32→1 Sigmoid.

## (b) Justification (fluorescence-specific)
1. Stride-2 conv + stacked convs = denoising in feature space (learned averaging suppresses zero-mean noise; maxpool would keep the brightest, often noise, pixel). 2. GroupNorm + percentile norm → intensity invariance (BatchNorm doubly wrong: tiny batches, leaks field-to-field brightness). 3. Depth gives receptive field spanning many object diameters while /16 preserves tiny structures. 4. Attention-gated skips judge features by learned context not raw magnitude, and clean boundary detail of its noise floor. 5. Bilinear upsampling keeps regression fields artifact-free.

## (c) Loss
Semantic (raw logits): L_sem = Tversky(α=0.3,β=0.7) + FocalBCE(γ=2.0,α_f=0.75). Tversky is area-invariant (gradient survives imbalance); β>α biases toward recovering faint objects; focal down-weights easy background. Instance heads (masked to foreground, boundary-weighted): flow weighted-MSE + cosine/direction term (at a seam the two objects' flows point opposite ways); distance weighted-MSE. Boundary weight w(x)=1+w0·exp(-(d1+d2)²/2σ²), w0=10, σ=3px (tighter than classic U-Net's 5 for dense small objects), peaks ~11× on the seam between the two nearest distinct instances, multiplies only regression terms. Total: λ_sem·L_sem + λ_inst·(5·L_flow+L_cos+L_dist); λ_flow=5 makes unit-vector MSE commensurable. Recommend warming up on the semantic head (λ_inst 0.1→1.0) or Kendall uncertainty weighting. All targets derivable from instance masks alone with zero intensity reads.

## (d) Postprocessing (non-learned, inference-time, distinct from loss)
1. Threshold foreground M=p_fg>0.5; opening + hole-fill. 2. Distance-peak seeds labeled as connected components (one marker per object interior — NOT raw peaks, which would over-split). 3. Flow-following: Euler-advect every foreground pixel along +flow to a sink; the marker it lands on assigns its ID. 4. Marker-controlled watershed on -d̂ as fallback. 5. Constrain to M (bg=0), drop tiny fragments, relabel. Primary over/under-seg knob: h/min_distance on the peak detector. HONEST LIMITATION: hard partition (one instance per pixel) — cannot represent TRUE multi-label overlap; acceptable only because 2D fluorescence "overlap" is treated as projection adjacency, not occlusion.

## (e) Reference PyTorch code (illustrative, NOT trained/benchmarked)
Model: ModifiedUNet with ConvBlock ([3×3 conv→GroupNorm→ReLU]×2), AttentionGate (alpha*x), symmetric encoder (stride-2 conv down)/decoder (bilinear up + conv), 4 attention-gated skips collected on the way down and merged on the way up, three 1×1-conv heads returning {"semantic","flow","distance"}. A `semantic_as_logits` flag reconciles the model (sigmoid in forward for inference) with the loss (raw logits for training). A `_match_size` helper pads/crops for non-power-of-two tiles. Loss: TverskyLoss + FocalBCELoss (nn.Modules), `instance_targets_and_weights` deriving fg/flow/distance/boundary-weight from a label map (requires scipy, no intensity reads), and CompositeInstanceLoss combining Tversky+Focal (foreground) with boundary-weighted flow-MSE + cosine + distance-MSE. Expected shapes: pred_fg_logits (B,1,H,W) raw logits, pred_flow (B,2,H,W) tanh, pred_dist (B,1,H,W) sigmoid; targets tgt_fg/tgt_flow/tgt_dist/w_bound. ## (f) Comparison (mechanistic only — no benchmark numbers claimed)
- StarDist: radial distances → star-convex polygon + NMS. Great for convex nuclei and intensity-invariant like ours, but the star-convexity prior clips irregular/branched shapes, ray count caps small-object detail, and object-level NMS can suppress dense neighbors. Our flow has no convexity prior and routes per-pixel, degrading gracefully at near-zero spacing.
- Cellpose: diffusion flow vectors → integrate to sinks. Closest baseline and genuinely strong — we RETAIN its flow mechanism and ADD a normalized-distance channel to seed/gate the decoding, anchoring basins with two agreeing geometric cues and giving tiny dim objects a stable interior peak where a flow-only sink can drift. This is an incremental robustness argument, not a categorical advantage.
- Mask R-CNN: RPN box proposals + per-ROI mask (RoIAlign). Uniquely can represent TRUE overlap (per-ROI masks may overlap in image space), which our dense prediction cannot. But anchor/proposal density misses small faint objects, box-NMS drops dense neighbors, RoIAlign loses tiny-object detail, box localization is the low-SNR bottleneck, and it needs box-level supervision. We are anchor-free, full-resolution, mask-only supervised — at the explicit cost of no true overlap.

## Caveats & open decisions (read before implementing)
1. **Unverified components:** hyperparameters (α/β, γ, w0=10, σ=3, λ_flow=5, depth=4) are sensible starting points to sweep, not tuned facts.
2. **The "overlap = projection adjacency, not occlusion" assumption is load-bearing** — the design produces a hard partition and cannot represent genuine multi-label overlap. If your data has true occluding objects, this is the wrong approach (use Mask R-CNN-style per-ROI masks). Confirm your regime first.
3. **Reconciled code inconsistency:** the model applies sigmoid to the semantic head; the loss expects raw logits — handled by the semantic_as_logits flag (True for training, default for inference). If you copy an earlier model without the flag, strip the sigmoid before training.
4. **Flow-derivation is a free parameter that must match end-to-end:** grad-of-smoothed-EDT (as coded) OR heat-diffusion-from-centroid — pick one, use it for tgt_flow, and match the postprocessing advection direction (follow +flow toward the interior sink). A mismatch silently breaks separation.
5. **Code is illustrative:** no training loop, data pipeline, augmentation, evaluation, or checkpointing; smoke tests verify only shapes/differentiability, not learning.

The novelty is honestly framed as a robustness-motivated superset of Cellpose (flow + an added distance-seed channel), not a claim of beating it — the intellectually honest posture when the contribution is an added signal on a strong existing mechanism.
