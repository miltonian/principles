# A Transformer-Bottleneck U-Net for Instance Segmentation of Noisy, Overlapping Fluorescence Microscopy Objects

## 0. Abstract

We present an end-to-end design for **instance segmentation** — assigning every foreground pixel to a specific object identity (cell A vs. cell B) — of noisy fluorescence microscopy images containing touching and overlapping cells or organelles. A plain foreground/background (*semantic*) mask cannot solve this task: touching objects fuse into one connected component. We therefore have the network predict an **instance-recoverable intermediate representation** — a three-channel tensor of foreground probability, a normalized per-instance distance transform, and boundary probability — using a **transformer-bottleneck U-Net** whose convolutional encoder/decoder denoise and preserve thin structures while a self-attention bottleneck supplies the global context needed to individuate overlapping objects. A compound, imbalance-aware loss supervises each channel; a marker-controlled watershed converts the representation into discrete integer **instance** labels, splitting touching objects at learned boundaries. We contrast the design mechanistically with Cellpose, StarDist, and Mask R-CNN. The design targets 2D single-plane images with a documented `Conv2d→Conv3d` path to 3D. All code is PyTorch and self-consistent: the model emits `[B, 3, H, W]` logits and the loss consumes exactly that shape and channel contract, and the postprocessing consumes exactly that per-channel semantics.

## 1. Problem framing

### 1a. Instance vs. semantic segmentation (definition)

**Instance segmentation** is the task of assigning every foreground pixel to a specific object identity — not merely calling a pixel "cell" but committing it to *this* cell (cell A) as distinct from the adjacent cell B. Its output is an integer-labeled map (or an equivalent set of per-object masks) in which each connected object receives a unique identifier, so that two touching or overlapping cells of the same class occupy disjoint, individually addressable masks. **Semantic segmentation** is strictly weaker: it partitions the image into class regions only — foreground vs. background (or class vs. class) — and is indifferent to how many distinct objects the foreground contains; a mask marking five touching organelles as "foreground" is semantically correct yet carries zero instance information. Instance segmentation therefore *subsumes* semantic segmentation (every instance labeling implies a foreground/background labeling) but adds object individuation. The two terms denote different tasks with different outputs and must never be used interchangeably.

The operational consequence dictates the rest of this document. A plain U-Net with a sigmoid/softmax foreground head is a *semantic* segmenter and cannot, by itself, separate touching cells — it must be augmented (boundary/border classes, distance-transform or gradient-flow regression, star-convex polygons, or region-proposal masks) plus a postprocessing step (watershed, peak-finding, flow-tracking) to recover instances. Evaluation must match: report instance-level scores (AP at IoU thresholds, Panoptic/Aggregated Jaccard) not pixel IoU/Dice alone, since a high semantic Dice can coexist with catastrophic under-segmentation of merged objects. Throughout, the per-pixel head is called *semantic* or *foreground* segmentation; the term *instance* is reserved for the identity-resolving end result produced only after postprocessing (Section 6). The subsumption is one-directional — a semantic loss or metric never validates instance performance.

### 1b. Scope: 2D single-plane, with a documented 3D extension path

**The objective specifies no dimensionality, so this design defaults to 2D single-plane fluorescence images**: a single H×W intensity plane, optionally multi-channel (stains, markers). The assumed input tensor convention is `[batch, channels, H, W]` (channels-first / PyTorch NCHW). The instance-segmentation output convention is a set of per-pixel dense maps of shape `[batch, K, H, W]` — a semantic/foreground map, a boundary map, and/or a distance-transform (or flow) map — which a separate postprocessing stage converts into an integer instance-label map `[batch, 1, H, W]` (0 = background, 1..N = object IDs). This scope fixes only these conventions; network internals, losses, and postprocessing are set below.

The 2D default is justified, not a limitation. **(a) Baseline parity:** the named comparators — Cellpose, StarDist, Mask R-CNN — all operate natively on 2D images, so matching their input shape keeps benchmarking apples-to-apples on the same data and metrics. **(b) Compute/memory economy:** 2D uses `Conv2d`, 2D pooling, and 2D attention, whose memory scales with H×W rather than D×H×W, so the denoising and touching-object separation mechanisms can be developed and validated first without volumetric overhead. **(c) No unrequested over-commitment:** the objective states no volumetric task, so defaulting to 3D would silently import anisotropic (z≠xy) voxel handling, z-resolution assumptions, and substantially larger annotation cost that were never asked for. **(d) Common acquisition case:** fluorescence data is frequently delivered as single optical planes or max-intensity projections, so 2D covers the dominant real-world input. This is discharging an unstated assumption, not narrowing the method.

*Out of scope for the 2D default (deferred, not forbidden):* volumetric receptive fields (3D convolutions), 3D connected-component instance recovery across z-slices, anisotropic z≠xy resampling, true 3D watershed/marker growing, and z-aware boundary/distance targets. **3D extension path.** When a volumetric task *is* specified, the same architecture generalizes by localized swaps: `Conv2d→Conv3d` (or pseudo-3D / anisotropic k×k×1 kernels for anisotropic voxels); input becomes `[B, C, D, H, W]` with output maps `[B, K, D, H, W]`; boundary and distance-transform targets and their losses generalize to 3D neighborhoods (26-connectivity distance transforms, 3D boundary surfaces); postprocessing generalizes to 3D watershed and 3D connected components over z-consistent connectivity. Main cost drivers to plan for: GPU memory (activations scale with D), patch-based volumetric training with overlap-tile stitching, and sparser, costlier 3D annotations. Keeping dimensionality behind a small set of layer/op abstractions (conv, pool, norm, distance-target, watershed) is exactly what makes the swap cheap and mechanical rather than a redesign; anisotropic-voxel handling is the likeliest friction point in any real 3D extension and should be flagged early.

### 1c. Assumptions (supervised learning)

We assume a fully **supervised** learning setting: the model learns exclusively from labeled examples and does not rely on self-supervised or unsupervised objectives. Concretely, we require a training set of fluorescence microscopy images paired with **per-instance segmentation annotations**, in which every pixel belonging to a cell or organelle is assigned to a specific object instance (instance label maps or per-object masks), *not* merely a binary foreground/background semantic mask that would leave touching objects indistinguishable.

We further assume these per-instance masks are **convertible into the dense, pixel-wise targets** the downstream network regresses or classifies — such as distance transforms toward object centers or boundaries, boundary/edge maps, and flow or gradient fields pointing to instance centroids. This convertibility is essential: it lets a U-Net, a dense pixel-prediction network, be trained for instance separation even though instance identities are never consumed directly as labels. As corollaries, we assume annotation quality and labeling consistency are sufficient, and that the labeled volume (optionally extended by augmentation) adequately covers the target domain. Generalization is therefore bounded by the labeled training distribution — its imaging modality, cell or organelle types, and noise characteristics. This scope is deliberately modality-agnostic: no dataset names, no image counts. If an augmentation or domain-adaptation stage is added later, it should build on, not contradict, the "generalization bounded by labeled distribution" clause.

## 2. Data characteristics

### 2a. Noise: signal-dependent Poisson + Gaussian (NOT pure additive Gaussian)

Noise in fluorescence microscopy is **not** well modeled as pure additive white Gaussian noise; it is fundamentally *signal-dependent*. The dominant term is Poisson-distributed photon shot noise arising from the quantized, stochastic arrival of photons at the detector: for a pixel collecting a mean of N photoelectrons, the count follows a Poisson law whose variance equals its mean (var = N), so the standard deviation scales as √N. On top of this sits an approximately signal-independent Gaussian read noise contributed by the detector electronics (amplifier and readout), plus a small dark-current term. The physically faithful model is therefore a mixed Poisson–Gaussian process,

```
    y = g · Poisson(x / g) + N(0, σ_read²)   (with optional offset/dark term),
```

where `x` is the underlying photon-rate signal and `g` is the camera gain. The practical consequence is *heteroscedasticity*: total noise variance rises with local intensity,

```
    var(y) ≈ g·x + σ_read²,
```

so bright structures carry more absolute noise while — decisively — dim structures carry the worst *relative* noise, because the signal-to-noise ratio behaves like √N and collapses as photon counts fall. In low-light live-cell and organelle imaging, faint boundaries, thin membranes, and the low-intensity "valleys" between touching or overlapping objects are precisely the regions with the fewest photons, hence the lowest SNR and the least reliable pixel evidence; read noise further dominates in these dim regimes where the shot-noise term is small. **Assuming stationary additive Gaussian noise here is not a harmless simplification: it mis-weights errors uniformly across intensities and understates uncertainty exactly where instance separation is hardest.**

**Concrete property downstream choices must answer to.** The noise is signal-dependent and heteroscedastic, driving relative SNR toward its minimum in dim, low-photon regions — i.e., at the faint boundaries and inter-object valleys where separation must occur. Later stages therefore must answer to this: the **architecture** should recover and amplify weak boundary/valley evidence under intensity-dependent noise (motivating variance-stabilizing preprocessing such as the generalized Anscombe transform to approximately equalize noise variance before the encoder, and attention/residual mechanisms that reweight low-SNR features rather than treat all pixels as equally trustworthy); the **loss** must not implicitly assume homoscedastic Gaussian error (plain per-pixel L2), and must place explicit emphasis on the low-SNR boundary/separation regions so that the sparse, noisy inter-object signal is not drowned out by abundant, higher-SNR interior pixels; **postprocessing** must derive seeds and separators from noise-tolerant representations rather than raw intensity minima, to avoid over-segmentation driven by shot-noise fluctuations in dim regions. Note that `g` and `σ_read` are camera-specific and often unknown, so variance stabilization is *motivated-but-optional*: a blind-noise or data-driven alternative may be needed when parameters are unavailable.

### 2b. Class imbalance

In these noisy images the pixel population is steeply skewed: background dominates the frame, foreground/interior (cell or organelle) pixels form a clear minority, and the inter-cell boundary/separation pixels that ridge the gap between touching instances are an **extreme minority** — typically only a thin 1–3 pixel-wide seam and, illustratively, well under ~1% of all pixels (background ≫ interior ≫ boundary; these figures are qualitative, not measured). This profile is intrinsic, not incidental: the objects are small and sparse, membranes are thin, and the very pixels that must separate overlapping instances are simultaneously the rarest and the most decision-critical.

The loss-design consequences are direct. Plain pixel-wise cross-entropy is dominated by abundant, easy background pixels, so a naive model minimizes loss by under-segmenting and merging adjacent objects while barely touching the separation ridge; the rarest class is exactly what instance splitting depends on, so neglecting it defeats separation; and global accuracy stays high while being misleading. This skew therefore motivates a family of remedies — class/pixel re-weighting, region-overlap objectives (Dice/Tversky), hard-example focusing (focal), and boundary-aware or distance-weighted terms that up-weight the thin separation pixels. The distance-weighted boundary term maps naturally to the classic U-Net weight map that up-weights touching-cell borders; the "merged instances" failure mode is the hook for the instance-splitting postprocessing (watershed / seeded separation) developed in Section 6. This imbalance hierarchy and boundary-criticality argument is the shared premise the loss and architecture sections cite when justifying compound losses and boundary-aware design choices.

## 3. Intermediate output representation — the shared channel contract

> **This section defines the canonical contract that Sections 4, 5, 6, and all code cite verbatim.**

### Why a plain binary mask fails

A plain binary foreground/background mask is *not* a valid output for this network and is explicitly rejected. Per the instance-vs-semantic distinction of Section 1a, a binary mask is a *semantic* artifact: it commits each pixel to "cell" vs. "not cell" but never to *which* cell. When two cells touch or overlap, their foreground regions fuse into a single connected component, and connected-component labeling — the only ID-recovery operation a binary mask supports — cannot split them; it emits one object where there are two. This failure is amplified by the noise profile of Section 2a: the low-photon inter-object "valley" that might have separated the instances is exactly where relative SNR collapses, so the thin separating gap is often eroded to nothing by shot noise. A binary head therefore structurally under-segments and merges adjacent instances. To individuate objects the network must instead predict an **intermediate representation whose *geometry* re-encodes identity**, from which a deterministic postprocessing step reconstructs discrete labels.

### Candidate representations

- **(a) Normalized per-instance distance transform (seed/energy landscape).** Each pixel stores its distance to the nearest boundary of *its own* instance, peaking at the object center. Recovery cue: interior peaks act as watershed seeds; per-object normalization forces a value dip-to-zero at every touching seam, manufacturing a separator even where intensity shows none. Failure mode: very elongated or ring-shaped objects have flat/degenerate ridges, needing a boundary channel to sharpen seams.
- **(b) Boundary/contour probability map.** Predicts the thin membrane ridge between objects. Recovery cue: subtract boundaries from foreground, then label. Failure mode: fragile under the Section 2a noise — boundaries are the lowest-SNR pixels (Section 2b), so gaps in a predicted contour re-merge instances; unusable *alone*, valuable as an auxiliary.
- **(c) Per-pixel flow / gradient-to-centroid field (Cellpose-style).** Each pixel predicts a 2-vector pointing toward its instance's seed; pixels following the flow converge into per-instance basins. Recovery cue: flow convergence clustering. Handles non-convex shapes well. Failure mode: heavier target construction (seed diffusion) and noisier convergence in dim regions.
- **(d) Star-convex radial distances (StarDist-style).** Each pixel emits distances to the object boundary along fixed rays. Recovery cue: polygon proposals + NMS. Failure mode: assumes **star-convexity** — poor for branched/non-convex organelles.

### Selected representation and exact target tensor format

We adopt a **normalized per-instance distance transform paired with a foreground channel and an auxiliary boundary channel**. The distance channel is robust to Section 2a noise because it is a smooth, region-integrated quantity (not a fragile 1-px edge), yet its per-object normalization guarantees a valley at every touching seam — solving the merge failure that kills binary masks. The foreground channel restores the semantic support (the Section 1a subsumption), and the boundary channel injects an explicit separator to sharpen seams where distance ridges are ambiguous. Distance+boundary handles moderately non-convex morphology better than star-convex radial distances; if organelles are strongly branched, the distance+boundary pair can be swapped locally for the 2-channel flow field (c) — the tensor spec is written so that substitution is local and leaves the encoder/bottleneck/skip design unchanged.

Because training is **supervised with per-instance annotations** (Section 1c), a ground-truth integer label map `L ∈ ℤ^{H×W}` (0 = background, 1..N = object IDs) converts deterministically into a target tensor of shape **`[C, H, W]` with `C = 3`** (batched: `[B, 3, H, W]`):

| Channel | Semantics | Target values | Activation | Derivation from `L` |
|---|---|---|---|---|
| **ch0** | foreground / semantic probability | {0,1} (soft [0,1] after augmentation) | sigmoid | `1[L>0]` |
| **ch1** | normalized per-instance distance transform | [0,1] | sigmoid | per-label EDT (0 at rim, growing inward), **max-normalized within each object** so each instance peaks at 1.0; background = 0 |
| **ch2** | boundary / contour probability | {0,1} | sigmoid | inner rim of each instance (EDT ≤ `boundary_width`), marking outer contours and, critically, touching seams |

*(Optional* `ch3` *of the **target** only — a precomputed U-Net separation weight map `w(x)`, consumed by the loss; see Section 5. It is never an output channel.)*

**Overlap/touching handling.** Annotations are a hard label map, so each pixel belongs to exactly one instance; the per-label EDT loop computes each object's transform independently, so a pixel adjacent to a neighbor still gets a *small* (near-zero) distance — the touching seam is thus a shared low ridge in ch1 and a marked ridge in ch2. If annotations encode genuine multi-ID overlap, assign each pixel to its nearest-centroid instance before conversion.

```python
import numpy as np
from scipy.ndimage import distance_transform_edt

def labels_to_targets(L: np.ndarray, boundary_width: int = 2) -> np.ndarray:
    """Integer instance-label map -> multi-channel target tensor.
    L: [H, W] int, 0=background, 1..N=object ids.
    Returns target [3, H, W] float32:
      ch0 = foreground prob {0,1}
      ch1 = per-instance normalized distance transform [0,1] (seams dip to 0)
      ch2 = boundary prob {0,1} (inner rim = separation seam)
    """
    H, W = L.shape
    fg = (L > 0).astype(np.float32)
    dist = np.zeros((H, W), np.float32)
    boundary = np.zeros((H, W), dtype=bool)
    for lbl in np.unique(L):
        if lbl == 0:
            continue
        mask = (L == lbl)
        edt = distance_transform_edt(mask)          # 0 at rim, grows inward
        peak = edt.max()
        if peak > 0:
            dist[mask] = edt[mask] / peak           # per-object max-normalization
        boundary |= mask & (edt <= boundary_width)  # inner rim = separation seam
    return np.stack([fg, dist, boundary.astype(np.float32)], axis=0)
```

**Loss-compatibility note.** ch0 (foreground) and ch2 (boundary) are **classification** targets (BCE/Dice; boundary heavily up-weighted per Section 2b); ch1 (distance) is a **regression** target (SmoothL1/L1) **masked to foreground** so abundant background zeros do not dominate the gradient. Channel order (0 = fg, 1 = distance, 2 = boundary) is load-bearing: postprocessing and loss cite these indices verbatim. One caveat for the loss stage: per-instance max-normalization can over-inflate a tiny object's peak relative to a large one; a global cap or clip is an option if that becomes a problem. *(Extension: for strongly branched, non-convex organelles, ch1+ch2 may be replaced locally by a 2-channel flow field — a regression swap that leaves the encoder/bottleneck/skip design unchanged. The canonical contract used everywhere in this document remains the 3-channel fg/distance/boundary tensor above.)*

## 4. Architecture — Transformer-Bottleneck U-Net *(Required Part a)*

**We adopt a transformer-bottleneck U-Net.** A convolutional encoder–decoder does the local work — attenuating the signal-dependent noise of Section 2a and preserving thin, small structures through high-resolution skips — while a self-attention bottleneck supplies the *global* receptive field needed to decide which pixels belong to cell A versus the overlapping cell B. This single variant answers all three problem properties by assigning each to a distinct component: **(a) noise robustness** to the variance-stabilized input plus local conv/pool averaging, **(b) small-object sensitivity** to full-resolution skip connections, and **(c) long-range context for overlapping cells** to bottleneck self-attention. *Why not an attention-gated U-Net:* its gates reweight skip features but never build a genuinely global token-to-token receptive field, so two large overlapping cells spanning the frame cannot be jointly individuated; we instead obtain feature reweighting cheaply via GroupNorm and fold global reasoning into the bottleneck.

**Input:** `[B, 1, 256, 256]`. `C_in = 1` single-channel fluorescence per the Section 1b NCHW convention. We assume the plane has been passed through a generalized Anscombe transform to approximately equalize variance across intensities — directly addressing property **(a)**, because the noise is signal-dependent Poisson photon-shot noise plus Gaussian read noise (Section 2a), *not* pure additive Gaussian, so stabilizing √N-scaling variance before the encoder makes downstream filters see a near-homoscedastic field. With four ×2 downsamplings, H and W must be divisible by **16**; 256×256 gives a 16×16 token grid.

**Encoder (4 stages).** Each stage is a double 3×3 conv block (Conv–GroupNorm(8)–GELU ×2); stages are separated by stride-2 max-pool. Channel widths **32 → 64 → 128 → 256**:
- `s0 = e0(x)` → `[B, 32, 256, 256]`
- `s1 = e1(pool s0)` → `[B, 64, 128, 128]`
- `s2 = e2(pool s1)` → `[B, 128, 64, 64]`
- `s3 = e3(pool s2)` → `[B, 256, 32, 32]`

Local 3×3 receptive fields plus pooling perform spatial averaging that suppresses per-pixel Poisson+Gaussian fluctuations — property **(a)**. GroupNorm (not BatchNorm) is chosen for stability under the heteroscedastic activations Section 2a warns about and small microscopy batch sizes — property **(a)**.

**Bottleneck.** `pool(s3)` → a 16×16 field; a `mid` conv block lifts it to `[B, 512, 16, 16]`. The 256 spatial locations are flattened to 256 tokens of dim 512, given learned positional embeddings, and passed through 4 pre-norm Transformer encoder layers (8-head self-attention, MLP ratio 4, GELU), then reshaped back to `[B, 512, 16, 16]`. Every token attends to every other, giving a receptive field covering the entire 256×256 input — property **(c)**: this is where the network reasons about the global arrangement of touching/overlapping instances that no stack of local convs can see at once. Positional embeddings preserve the spatial relations needed for that reasoning.

**Skip connections.** Identity concatenation `e0→d0`, `e1→d1`, `e2→d2`, `e3→d3` (encoder stage → symmetric decoder stage). The full-resolution `s0` skip (256×256) is the load-bearing one for property **(b)**: the boundary channel (ch2) is an extreme-minority 1–3 px seam (Section 2b), and only an un-downsampled path can carry those thin separators and small organelles intact to the head.

**Decoder (4 stages).** Each stage: bilinear upsample ×2 + 3×3 conv (chosen over transpose conv to avoid checkerboard artifacts that would masquerade as spurious boundary/noise — property **(a)**), concatenate the matching skip, then a double conv block. Widths mirror the encoder **256 → 128 → 64 → 32**, restoring `[B, 32, 256, 256]`.

**Output head.** A 1×1 conv `32 → 3`, emitting **`[B, 3, 256, 256]`** logits, with a per-channel sigmoid (all three targets lie in [0,1]): **ch0 = foreground/semantic probability, ch1 = normalized per-instance distance transform, ch2 = boundary probability** — exactly the Section 3 contract. This is emphatically *not* a plain binary mask: ch1's per-object-normalized valleys and ch2's seams are what let downstream postprocessing recover discrete instances. `C_out = 3` matches the contract verbatim. The head returns logits; the loss uses BCE-with-logits on ch0/ch2 and sigmoid+masked-regression on ch1 — do not double-apply the sigmoid.

| Architectural choice | Property | Rationale |
|---|---|---|
| Anscombe variance-stabilized input | (a) | Equalizes √N Poisson-shot + Gaussian-read variance (Section 2a) before the encoder |
| 3×3 conv blocks + max-pool encoder | (a) | Local averaging attenuates signal-dependent Poisson+Gaussian noise |
| GroupNorm (not BatchNorm) | (a) | Stable under heteroscedastic activations and small batches |
| 4-stage downsampling to 16×16 | (c) | Enlarges receptive field / produces a compact token grid for attention |
| Transformer bottleneck self-attention | (c) | Global token-to-token context to individuate overlapping cells |
| Learned positional embeddings | (c) | Preserve spatial relations for reasoning about cell arrangement |
| Full-res `e0→d0` (and all) skips | (b) | Preserve thin boundaries (ch2) and small objects lost to downsampling |
| Bilinear-upsample + conv decoder | (a) | Avoids checkerboard artifacts that mimic noise/false boundaries |
| 3-channel sigmoid head (fg/dist/boundary) | (b)+(c) | Encodes recoverable instance geometry (Section 3); not a binary mask |

A note on variable input sizes: 256×256 is chosen for concreteness; any H, W divisible by 16 works, but the transformer's learned positional embedding is tied to the 16×16 token count — variable input sizes need interpolated or relative position encodings.

## 5. Loss functions *(Required Part b)*

The loss binds to the Section 3 contract: the model emits `pred ∈ [B, 3, H, W]` **logits**, and the ground-truth target is `target ∈ [B, 3, H, W]` (optionally `[B, 4, H, W]` with a precomputed per-pixel boundary weight map `w(x)` in ch3). Because the head returns logits, classification terms use `binary_cross_entropy_with_logits` (numerically stable), while Dice/Tversky/focal/regression terms apply `sigmoid` first — we never double-apply the sigmoid. The output is **instance-recoverable geometry, not a semantic mask**: at least one term (the masked distance regression on ch1) *directly* supervises the instance-separating channel, so no single BCE-on-a-mask term appears in isolation.

### 5.1 Loss-term → output-channel mapping

| # | Term | Channel | Loss family | Justification (semantics + imbalance) |
|---|---|---|---|---|
| L₁ | Soft Dice | ch0 fg | region-overlap | Dice is a set-overlap *ratio*, intrinsically invariant to the background/foreground pixel ratio → survives foreground-minority skew (2b) where mean-BCE collapses to "all background". |
| L₂ | Tversky (α=0.3, β=0.7) | ch0 fg | asymmetric overlap | β>α penalizes false negatives (missed faint cells) harder than false positives → recall-favoring for the minority foreground. |
| L₃ | Focal BCE (γ=2) | ch0 fg | hard-example focusing | γ down-weights the abundant, easy, high-SNR background so gradient mass flows to hard low-SNR interior pixels (2a/2b). |
| L₄ | Weighted boundary BCE | ch2 boundary | spatially-weighted focal BCE | The separating seam is an extreme minority (<~1%, 2b); a U-Net `w(x)` map up-weights the thin membrane between *touching* objects by ~an order of magnitude so it is not drowned out by a raw pixel average. |
| L₅ | Focal-Tversky (α=0.3, β=0.7) | ch2 boundary | asymmetric overlap + focal | Complementary boundary-*recall* pressure: a *broken* predicted seam re-merges instances, so any gap must be penalized directly. |
| L₆ | Masked SmoothL1 | ch1 distance | regression | Directly supervises the instance-separating geometry — accurate per-object distance peaks/valleys are what watershed seeds and splits touching cells; masked to foreground since background has no meaningful distance target. |

Every one of the three output channels is covered by ≥1 term (ch0: L₁–L₃; ch1: L₆; ch2: L₄–L₅).

`★ Insight ─────────────────────────────────────`
**Why Dice/Tversky are inherently imbalance-robust.** Pixel-averaged BCE sums a per-pixel penalty, so if ~98% of pixels are easy background, the background gradient dominates and the loss is minimized by predicting "background everywhere" — precisely the under-segmentation failure of Section 2b. Dice instead measures `2·|A∩B| / (|A|+|B|)`, a *ratio* whose value depends only on overlap between the (small) predicted and (small) true foreground sets; adding more true-negative background pixels does **not** change it. That scale-invariance is what makes it the right base term for the foreground-minority channel — no per-class weight tuning required.
`─────────────────────────────────────────────────`

### 5.2 Foreground / region losses (foreground-minority imbalance)

The foreground channel (ch0) is a clear minority. We supervise it with a **Dice + Tversky + focal** combination rather than plain BCE. Under a 50:1 (or worse) background:foreground ratio, the arithmetic mean of per-pixel BCE is dominated by the many easy, correctly-classified background pixels, and the optimizer drives foreground logits down to shave the last bits of background loss, yielding high pixel-accuracy but systematic under-segmentation. **Soft Dice (L₁)** provides the frequency-invariant backbone. **Tversky (L₂)** generalizes Dice with an asymmetric FP/FN trade-off, `T = TP / (TP + α·FP + β·FN)`; we set α = 0.3, β = 0.7 so false negatives (missing a dim cell — the Section 2a low-SNR failure) cost more than false positives, deliberately biasing toward recall of the minority foreground. **Focal BCE (L₃)** with γ = 2.0 multiplies each pixel's BCE by `(1 − p_t)^γ`, shrinking the loss on confidently-correct easy background toward zero and concentrating capacity on hard, ambiguous low-SNR pixels while still supplying a calibrated per-pixel probabilistic gradient.

### 5.3 Dedicated boundary / contour loss (extreme-minority seam)

The boundary channel (ch2) is the **extreme minority** — a 1–3 px seam, illustratively well under ~1% of pixels — yet it is the single most decision-critical class for splitting touching/overlapping objects. A raw pixel-averaged BCE would treat these seam pixels as statistical noise. We give them dedicated, imbalance-robust treatment via two terms.

**(a) Spatially-weighted focal BCE (L₄) with a U-Net weight map.** We revive the original U-Net separation weight map `w(x) = w_c(x) + w₀ · exp(−(d₁(x)+d₂(x))² / (2σ²))`, where `d₁`, `d₂` are the distances from pixel `x` to the borders of the **nearest** and **second-nearest** instances. Where two objects nearly touch, `d₁+d₂` is small, the exponential saturates, and the pixel receives a weight up to `w₀` (recommended **w₀ = 10, σ = 5 px**) on top of a class-balance term `w_c`. This is exactly what rescues touching/overlapping objects: the pixels in the thin valley *between* two cells — the ones that must fire to separate cell A from cell B — get an order-of-magnitude larger gradient than interior or background pixels, so the network is forced to learn the separator instead of averaging it away. The weight map is deterministically derived from the instance label map and best precomputed at target-generation time and passed as `target[:, 3]`; if absent, the code falls back to a per-batch positive/negative inverse-frequency weight.

**(b) Focal-Tversky on the boundary (L₅).** A weighted-BCE alone still optimizes a per-pixel objective. We add a Focal-Tversky term (α = 0.3, β = 0.7, focal exponent 4/3) on ch2 to add an *overlap*-based, recall-favoring pressure: it directly penalizes a fragmented or partially-missing predicted seam (high FN), because any gap in the contour lets two instances re-merge downstream.

### 5.4 Masked regression for the geometry channel

The distance channel (ch1) is a **regression** target and is the term that most directly encodes instance-separating geometry. We use **SmoothL1 / Huber** (robust to the heavy-tailed residuals expected under Section 2a noise, and less peaky than MSE near the seam) between `sigmoid(pred[:,1])` and the normalized EDT target, **masked to foreground** (`target[:,0] > 0.5`):

```
L₆ = Σ_x m(x)·SmoothL1(d̂(x), d(x)) / (Σ_x m(x) + ε),   m = foreground mask
```

`★ Insight ─────────────────────────────────────`
**Why the geometry loss must be masked.** Background pixels have **no defined distance-to-own-boundary** — Section 3 encodes them as 0 by convention, but that 0 is a *placeholder*, not a learning signal. If we regressed over all pixels, the abundant background zeros (the same 2b majority) would dominate the SmoothL1 average and the network would learn "predict ~0 everywhere," flattening the very interior peaks that watershed needs as seeds. Masking to foreground restricts the regression to pixels where the target is meaningful, and dividing by the mask area (not the image area) keeps the term's scale stable across images with different foreground fractions.
`─────────────────────────────────────────────────`

Because Section 3 max-normalizes the EDT per object, every instance peaks at 1.0 regardless of size, so the regression is scale-invariant across large and small cells (small organelles are not down-weighted). *Flow substitution:* if ch1+ch2 are replaced by a 2-channel `(dy, dx)` flow field, L₆ is replaced by masked MSE on the vector field plus a cosine/angular term `1 − cos(∠(v̂, v))`, both masked to foreground — MSE fixes magnitude, the cosine term fixes direction (critical because postprocessing follows flow *direction* to converge pixels onto seeds).

### 5.5 Composite total loss

We combine the terms as an explicit weighted sum, boundary/geometry deliberately up-weighted so those terms — which carry the instance information — are not swamped by the fg terms:

```
L_total = 1.0·L_Dice(ch0) + 0.5·L_Tversky(ch0) + 0.5·L_Focal(ch0)
        + 2.0·L_wBoundary(ch2) + 1.0·L_FocalTversky(ch2) + 1.0·L_MaskedDist(ch1)
```

**Balancing heterogeneous scales.** Dice/Tversky terms are bounded in [0,1]; the weighted-BCE term can be much larger because of the `w₀=10` map; SmoothL1 on [0,1] targets is O(0.1). Three practical strategies, in increasing sophistication: (i) the fixed weighting above, after a one-epoch sanity check that each *weighted* term lands within ~one order of magnitude in the logged per-term dict; (ii) normalize each term by a running EMA of its own magnitude so no term dominates by units alone; (iii) learned homoscedastic-uncertainty weighting (Kendall & Gal) or GradNorm, replacing each `wᵢ` with `1/(2σᵢ²) + log σᵢ` and learning `σᵢ`. The imbalance argument dictates the *floor*: the boundary (L₄, L₅) and geometry (L₆) terms must retain enough effective weight that their gradients are not overwhelmed by the fg terms.

The Tversky/Focal-Tversky asymmetry (`β` for false-negative penalty) and `w_boundary`/`w₀` are the two knobs that most change behavior. Higher `β` (→0.8) and higher `w_boundary` push toward **over-segmentation** (aggressive splitting, good when merges are the dominant error); lower values push toward **under-segmentation** (fewer false splits, good when your seams are noisy). Tune on an *instance-level* metric (AP@IoU / Panoptic Quality), **not** pixel Dice, since pixel metrics are blind to the merge/split errors you care about.

## 6. Postprocessing — representation → discrete instance labels *(Required Part c)*

This stage converts the network's intermediate representation into a discrete integer **instance-label** map. It **consumes the exact Section 3 output tensor** — it never thresholds a plain foreground mask, which would merge every touching pair into one connected component.

**Inputs.** One tensor `output` of shape `[3, H, W]` (or `[1, 3, H, W]`; batch dim squeezed), dtype float, all values in `[0, 1]` (post-sigmoid): channel [0] = foreground/semantic probability; channel [1] = normalized per-instance distance transform (each interior peaks ~1.0, every touching seam dips toward 0); channel [2] = boundary probability (the inner rim / inter-object seam). Assumption: values are already post-sigmoid; if raw logits are passed, apply a sigmoid first. This is the distance-transform representation family, so the primary method is **local-peak seed detection + marker-controlled watershed** (flow-tracking and polygon-NMS are the methods for the flow and star-convex representations respectively, which Section 3 did not select).

**Step-by-step algorithm.**
1. **Foreground support.** Threshold channel 0: `foreground = (ch0 ≥ fg_threshold)`. This is the domain over which instances may exist and the outer boundary of the watershed.
2. **Smooth the distance map** (noise-tolerant seeding). Gaussian-filter channel 1 with `smoothing_sigma`, then multiply by `foreground`. Smoothing removes shot-noise-driven false local maxima before peak finding — seeds are taken from a *noise-tolerant*, region-integrated quantity, never raw intensity minima.
3. **Seed / marker detection.** Find local maxima of the smoothed distance map with `peak_local_max`, requiring value `≥ peak_threshold` and separation `≥ min_peak_distance`, restricted to `foreground`. Reject any peak whose boundary probability (channel 2) exceeds `boundary_threshold` — a true interior seed should not sit on a predicted seam. Label the surviving peaks with `ndi.label` to get integer markers, one per intended instance.
4. **Marker-controlled watershed.** Build the elevation surface `elevation = -dist_s + ch2`: watershed descends into distance peaks and *climbs over* the predicted boundary ridge. Flood from the markers with `mask = foreground`. Each marker grows a basin; basins meet exactly at the low-distance / high-boundary seam.
5. **Min-size filtering.** Remove instances smaller than `min_object_size` pixels (noise-induced fragments), then relabel contiguously to `1..N` (0 = background). Return `int32`.

*(Fallback: if seeds are unstable in very dim fields, replace peak seeding with h-maxima of the distance map — `skimage.morphology.h_maxima` — which is more robust to shallow noise peaks at the cost of one extra `h` parameter.)*

| Hyperparameter | Default | Sweep | Trades off |
|---|---|---|---|
| `fg_threshold` | 0.5 | 0.3–0.7 | Lower recovers faint/dim objects (low-SNR, 2a) but admits noise as foreground; higher tightens masks but erodes dim rims. |
| `peak_threshold` | 0.5 | 0.3–0.7 | Lower yields more seeds (over-seg risk); higher yields fewer seeds (merges under-seeded cells). |
| `min_peak_distance` | 5 | 3–15 | Smaller separates small adjacent cells but may double-seed one; larger suppresses duplicates but can merge nearby objects. |
| `boundary_threshold` | 0.5 | 0.3–0.7 | Lower aggressively rejects seam seeds and raises barriers; higher effectively ignores ch2. |
| `smoothing_sigma` | 1.0 | 0.0–3.0 | Larger suppresses noise-driven spurious peaks but can blur two close peaks into one; smaller keeps sharp seeds but is noise-sensitive. |
| `min_object_size` | 25 | 5–200 | Larger removes noise fragments but can delete genuinely tiny organelles; smaller retains small objects but retains noise blobs. |

**How touching / overlapping objects are resolved (the crux).** The separation power comes from the **per-instance normalization of channel 1**, not from any raw-intensity gap. Two touching cells each get their own distance transform that peaks at their own center and decays to ~0 at their shared rim. Consequently the smoothed distance map has **two distinct maxima with a valley between them** — even where the fluorescence intensity shows no gap because the low-photon inter-object valley was eroded by shot noise (2a). Step 3 therefore places **two markers**, and marker-controlled watershed grows **two basins** that collide precisely at the valley: two instances where a binary mask + connected-components would emit one. The boundary channel makes the cut deterministic at the interface: adding ch2 to the elevation raises a ridge exactly along the predicted seam, so the watershed line snaps to the learned membrane rather than wandering. For **genuine overlap/occlusion**, where one object's pixels are physically shared, the hard label map assigns each pixel to a single instance (nearest-centroid at encoding time); postprocessing then yields a mutually-exclusive partition and cannot represent co-located masks — a concession revisited in Section 8. (True amodal/overlapping masks would require a per-instance overlap channel, which is out of scope for this 3-channel representation.)

**Noise handling.** The pipeline suppresses spurious instances from Poisson+Gaussian noise at three points. **Distance smoothing** (`smoothing_sigma`) removes the shallow, high-frequency local maxima that shot noise injects into dim regions, so seeds are drawn from a noise-tolerant, region-integrated quantity rather than raw pixel minima. **Probability thresholding** (`fg_threshold`, `peak_threshold`, `boundary_threshold`) discards low-confidence foreground and seed candidates, exactly the low-SNR pixels where the network is least certain. **Min-size filtering** (`min_object_size`) deletes the few surviving noise-born fragments. Together these enforce that a candidate instance must present a smooth, sufficiently tall, sufficiently large distance peak inside confident foreground — a bar that Poisson/Gaussian fluctuations in low-photon regions rarely clear.

## 7. Code — model + composite loss (one PyTorch stack) *(Required Part d)*

Both blocks form a single self-consistent stack: the model emits `[B, 3, H, W]` logits (ch0 fg, ch1 distance, ch2 boundary); the loss consumes predictions of exactly that shape and targets in the Section 3 format. `model(x) → CompositeLoss(pred, target)` runs without shape mismatch, and an optional postprocessing block (Section 6) matches the same channel contract.

### 7.1 Model

```python
import torch, torch.nn as nn

class ConvBlock(nn.Module):
    """2x (3x3 conv, GroupNorm, GELU). Local averaging attenuates
    Poisson+Gaussian noise (a); GroupNorm is batch-size robust."""
    def __init__(self, cin, cout):
        super().__init__()
        self.net = nn.Sequential(
            nn.Conv2d(cin, cout, 3, padding=1), nn.GroupNorm(8, cout), nn.GELU(),
            nn.Conv2d(cout, cout, 3, padding=1), nn.GroupNorm(8, cout), nn.GELU())
    def forward(self, x): return self.net(x)

class TransformerBottleneck(nn.Module):
    """Global self-attention over the 16x16 token grid -> long-range
    context (c) for individuating overlapping cells."""
    def __init__(self, ch, tokens=256, heads=8, layers=4):
        super().__init__()
        self.pos = nn.Parameter(torch.zeros(1, tokens, ch))
        layer = nn.TransformerEncoderLayer(ch, heads, ch * 4, batch_first=True,
                                           activation='gelu', norm_first=True)
        self.tf = nn.TransformerEncoder(layer, layers)
    def forward(self, x):
        B, C, H, W = x.shape                              # (B, 512, 16, 16)
        t = x.flatten(2).transpose(1, 2) + self.pos       # (B, H*W, C)
        return self.tf(t).transpose(1, 2).reshape(B, C, H, W)

class Up(nn.Module):
    """Bilinear upsample + conv (no checkerboard, (a)) then fuse skip (b)."""
    def __init__(self, cin, cskip, cout):
        super().__init__()
        self.up = nn.Sequential(
            nn.Upsample(scale_factor=2, mode='bilinear', align_corners=False),
            nn.Conv2d(cin, cout, 3, padding=1))
        self.fuse = ConvBlock(cout + cskip, cout)
    def forward(self, x, skip):
        return self.fuse(torch.cat([self.up(x), skip], dim=1))

class TransformerBottleneckUNet(nn.Module):
    def __init__(self, c_in=1, c_out=3, w=(32, 64, 128, 256, 512)):
        super().__init__()
        self.e0, self.e1 = ConvBlock(c_in, w[0]), ConvBlock(w[0], w[1])
        self.e2, self.e3 = ConvBlock(w[1], w[2]), ConvBlock(w[2], w[3])
        self.pool = nn.MaxPool2d(2)
        self.mid = ConvBlock(w[3], w[4])
        self.tf  = TransformerBottleneck(w[4], tokens=16 * 16)
        self.d3, self.d2 = Up(w[4], w[3], w[3]), Up(w[3], w[2], w[2])
        self.d1, self.d0 = Up(w[2], w[1], w[1]), Up(w[1], w[0], w[0])
        self.head = nn.Conv2d(w[0], c_out, 1)   # ch0 fg, ch1 dist, ch2 boundary
    def forward(self, x):                        # x: (B, 1, 256, 256) Anscombe-stabilized
        s0 = self.e0(x)                          # (B, 32, 256, 256)
        s1 = self.e1(self.pool(s0))              # (B, 64, 128, 128)
        s2 = self.e2(self.pool(s1))              # (B, 128, 64, 64)
        s3 = self.e3(self.pool(s2))              # (B, 256, 32, 32)
        b  = self.tf(self.mid(self.pool(s3)))    # (B, 512, 16, 16) global attn (c)
        x  = self.d3(b, s3)                       # (B, 256, 32, 32)
        x  = self.d2(x, s2)                       # (B, 128, 64, 64)
        x  = self.d1(x, s1)                       # (B, 64, 128, 128)
        x  = self.d0(x, s0)                       # (B, 32, 256, 256)
        return self.head(x)                       # (B, 3, 256, 256) LOGITS -> per-ch sigmoid

if __name__ == "__main__":
    m = TransformerBottleneckUNet()
    y = m(torch.randn(2, 1, 256, 256))            # (B, 1, H, W)
    assert y.shape == (2, 3, 256, 256), y.shape   # matches the Section 3 contract
    print("model OK:", tuple(y.shape))
```

### 7.2 Composite loss

```python
import numpy as np
import torch
import torch.nn as nn
import torch.nn.functional as F
from scipy.ndimage import distance_transform_edt   # offline weight-map builder only

EPS = 1e-6

# --- foreground / region losses, one channel each: (B, H, W) ---------------
def soft_dice_loss(logit, target):                 # logit/target: (B, H, W)
    p = torch.sigmoid(logit)                        # head emits logits
    inter = (p * target).sum(dim=(1, 2))            # (B,)
    denom = p.sum(dim=(1, 2)) + target.sum(dim=(1, 2))
    return (1.0 - (2 * inter + EPS) / (denom + EPS)).mean()

def tversky_loss(logit, target, alpha=0.3, beta=0.7, focal_gamma=1.0):
    p = torch.sigmoid(logit)                        # (B, H, W)
    tp = (p * target).sum(dim=(1, 2))
    fp = (p * (1 - target)).sum(dim=(1, 2))
    fn = ((1 - p) * target).sum(dim=(1, 2))
    t = (tp + EPS) / (tp + alpha * fp + beta * fn + EPS)   # beta>alpha -> recall bias
    return ((1.0 - t) ** focal_gamma).mean()

def focal_bce_loss(logit, target, alpha=0.25, gamma=2.0):
    bce = F.binary_cross_entropy_with_logits(logit, target, reduction="none")
    p = torch.sigmoid(logit)
    p_t = p * target + (1 - p) * (1 - target)
    a_t = alpha * target + (1 - alpha) * (1 - target)
    return (a_t * (1 - p_t) ** gamma * bce).mean()

# --- boundary weight map (offline) + weighted boundary loss ----------------
def make_boundary_weight_map(label_map, w0=10.0, sigma=5.0, wc=1.0):
    """U-Net separation map from instance labels (0=bg). Returns w(x) [H, W]."""
    H, W = label_map.shape
    ids = [i for i in np.unique(label_map) if i != 0]
    w = np.full((H, W), wc, dtype=np.float32)
    if len(ids) >= 2:
        dists = np.stack([distance_transform_edt(label_map != i) for i in ids], 0)
        dists.sort(axis=0)
        d1, d2 = dists[0], dists[1]                 # nearest, 2nd-nearest borders
        w = wc + w0 * np.exp(-((d1 + d2) ** 2) / (2.0 * sigma ** 2))
    return w.astype(np.float32)

def weighted_boundary_loss(logit, target, weight_map=None, gamma=2.0):
    bce = F.binary_cross_entropy_with_logits(logit, target, reduction="none")
    p = torch.sigmoid(logit)
    focal = (1 - (p * target + (1 - p) * (1 - target))) ** gamma
    if weight_map is None:                          # per-batch inverse-freq fallback
        pos = target.sum() + EPS
        neg = (1 - target).sum() + EPS
        weight_map = target * (neg / pos) + (1 - target) * 1.0
    return (weight_map * focal * bce).mean()

# --- masked distance regression (instance geometry) ------------------------
def masked_distance_loss(logit, target_dist, fg_mask, beta=1.0):
    pred = torch.sigmoid(logit)                     # (B, H, W)
    per_px = F.smooth_l1_loss(pred, target_dist, beta=beta, reduction="none")
    m = (fg_mask > 0.5).float()                     # background has no distance target
    return (per_px * m).sum() / (m.sum() + EPS)

class CompositeLoss(nn.Module):
    """Consumes head logits (B,3,H,W) + Section-3 target (B,3,H,W [+w-map])."""
    def __init__(self, w=None, tversky=(0.3, 0.7)):
        super().__init__()
        self.w = w or dict(dice=1.0, tversky=0.5, focal=0.5,
                           wbnd=2.0, ftversky=1.0, dist=1.0)
        self.alpha, self.beta = tversky
    def forward(self, pred, target):
        # pred:   (B, 3, H, W) logits;  target: (B, 3, H, W) (+ optional (:, 3) w-map)
        fg_l, dist_l, bnd_l = pred[:, 0], pred[:, 1], pred[:, 2]      # each (B, H, W)
        fg_t, dist_t, bnd_t = target[:, 0], target[:, 1], target[:, 2]
        wmap = target[:, 3] if target.shape[1] > 3 else None
        L = {
            "dice":     soft_dice_loss(fg_l, fg_t),                   # ch0 (2b)
            "tversky":  tversky_loss(fg_l, fg_t, self.alpha, self.beta),
            "focal":    focal_bce_loss(fg_l, fg_t, gamma=2.0),
            "wbnd":     weighted_boundary_loss(bnd_l, bnd_t, wmap),   # ch2 (2b)
            "ftversky": tversky_loss(bnd_l, bnd_t, self.alpha, self.beta, focal_gamma=4/3),
            "dist":     masked_distance_loss(dist_l, dist_t, fg_mask=fg_t),  # ch1 geometry
        }
        total = sum(self.w[k] * v for k, v in L.items())
        return total, {k: v.detach() for k, v in L.items()}

if __name__ == "__main__":
    from math import isfinite
    B, H, W = 2, 64, 64
    pred = torch.randn(B, 3, H, W, requires_grad=True)               # head output
    lbl = np.zeros((H, W), np.int32); lbl[10:30, 10:30] = 1; lbl[10:30, 30:50] = 2
    fg = (lbl > 0).astype(np.float32); dist = np.zeros((H, W), np.float32)
    for i in (1, 2):
        e = distance_transform_edt(lbl == i)
        if e.max() > 0: dist[lbl == i] = e[lbl == i] / e.max()
    bnd = ((lbl > 0) & (distance_transform_edt(lbl > 0) <= 2)).astype(np.float32)
    wmap = make_boundary_weight_map(lbl)
    tgt = torch.from_numpy(np.stack([fg, dist, bnd, wmap], 0)[None].repeat(B, 0)).float()
    total, logs = CompositeLoss()(pred, tgt)                          # pred/tgt: (B,*,H,W)
    total.backward()
    assert isfinite(float(total)) and pred.grad is not None
    print("loss OK:", round(float(total), 4), {k: round(float(v), 3) for k, v in logs.items()})
```

### 7.3 Postprocessing (optional, matches Section 6)

```python
import numpy as np
from scipy import ndimage as ndi
from skimage.feature import peak_local_max
from skimage.segmentation import watershed
from skimage.morphology import remove_small_objects

def postprocess_to_instances(output, fg_threshold=0.5, peak_threshold=0.5,
                             min_peak_distance=5, boundary_threshold=0.5,
                             smoothing_sigma=1.0, min_object_size=25):
    """output: [3,H,W] or [1,3,H,W] post-sigmoid in [0,1]
       (ch0 fg, ch1 distance, ch2 boundary). Returns int32 [H,W], bg=0."""
    arr = np.asarray(output, dtype=np.float32)
    if arr.ndim == 4: arr = arr[0]
    fg_prob, dist, bnd_prob = arr[0], arr[1], arr[2]
    foreground = fg_prob >= fg_threshold                             # Step 1
    dist_s = ndi.gaussian_filter(dist, sigma=smoothing_sigma) * foreground  # Step 2
    peaks = peak_local_max(dist_s, min_distance=min_peak_distance,   # Step 3
                           threshold_abs=peak_threshold, labels=foreground,
                           exclude_border=False)
    seed_mask = np.zeros(dist_s.shape, dtype=bool)
    if len(peaks):
        pr, pc = peaks[:, 0], peaks[:, 1]
        keep = bnd_prob[pr, pc] < boundary_threshold                 # reject seam seeds
        seed_mask[pr[keep], pc[keep]] = True
    markers, _ = ndi.label(seed_mask)
    labels = watershed(-dist_s + bnd_prob, markers, mask=foreground) # Step 4
    labels = remove_small_objects(labels, min_size=min_object_size)  # Step 5
    out = np.zeros(labels.shape, dtype=np.int32)
    for new_id, old in enumerate(np.unique(labels[labels > 0]), start=1):
        out[labels == old] = new_id
    return out
```

## 8. Comparison to existing models *(Required Part e)*

All four approaches — Cellpose, StarDist, Mask R-CNN, and the proposed design — are competent *semantic* segmenters; what fundamentally distinguishes them is **how they individuate touching or overlapping instances** (the cell-A-vs-cell-B problem of Section 1a). Each commits to a different intermediate representation and a different mechanism for turning that representation into discrete object identities, and each mechanism has characteristic failure modes precisely in the regime that matters here: **noisy (signal-dependent Poisson shot + Gaussian read, 2a), densely packed, overlapping fluorescence objects** whose separating seams are the lowest-SNR, extreme-minority pixels (2b). For reference, the **proposed design** predicts the Section 3 representation (fg / normalized per-instance distance / boundary) with a transformer-bottleneck U-Net (Section 4) and recovers instances by seed detection + marker-controlled watershed (Section 6). A watershed "marker" is a labeled seed region from which a basin grows; NMS (non-maximum suppression) is the greedy removal of lower-scoring candidates that overlap a higher-scoring one.

| Model | Separation mechanism / representation | How touching objects split | Overlap handling | Noise robustness (Poisson+Gaussian) | Small-object sensitivity | Key limitation here |
|---|---|---|---|---|---|---|
| **Cellpose** | Per-pixel **flow field** to cell-center sinks + foreground | Follow/integrate flows; pixels to same sink = one object | No amodal overlap; non-convex OK | Flow *direction* degrades in dim regions | Good; tiny-cell sinks noisy | Co-located objects can't both own a pixel |
| **StarDist** | **Star-convex polygon distances** + object prob | NMS over candidate polygons | Convex-only; NMS drops overlaps | Prob map fairly noise-tolerant | Strong for round nuclei | Rays can't represent concavities / occluded centers |
| **Mask R-CNN** | **Region proposals + per-ROI mask heads** (RPN, RoIAlign) | Each ROI segments independently after box-NMS | **Can represent overlap** | Box detection degrades for faint objects | Weak: small clustered objects evade proposals | RPN/RoIAlign miss dense faint touching cells |
| **Proposed** | **Normalized per-instance distance + boundary + fg**, transformer-bottleneck U-Net | Distance-peak seeds → watershed split at boundary ridge | Hard partition (no amodal); non-convex OK | Explicitly tuned: Anscombe, GroupNorm, boundary up-weighting | Strong: full-res skips + attention preserve seams | Watershed can under-segment genuine overlap |

**Cellpose** predicts a **spatial gradient flow field**: for every foreground pixel it regresses a 2-vector pointing "uphill" toward the center (sink) of the cell that pixel belongs to, together with a foreground probability. Instances are recovered by Euler-integrating each pixel along the predicted flow for a fixed number of steps; pixels that converge to a common sink are grouped into one object. Because grouping is by dynamical convergence rather than by shape priors, Cellpose handles arbitrarily non-convex, elongated, and branched morphologies and separates densely packed cells cleanly. But flow *direction* is exactly the quantity that degrades under heavy Poisson/Gaussian noise — in the dim inter-object valleys (2a) the estimated vectors become unreliable, so integration paths wander and either merge adjacent sinks or spawn spurious ones. More fundamentally, flow integration assigns each pixel to exactly one sink, so two genuinely *overlapping* (co-located) objects cannot both claim the same pixel; Cellpose partitions, it does not overlay.

**StarDist** predicts, for each pixel, a set of **star-convex polygon distances** — the radial distance to the object boundary along a fixed set of angles — plus an object-probability map. Instance candidates are polygons anchored at high-probability pixels, and **non-maximum suppression** resolves them: the highest-probability polygon is kept and overlapping lower-scoring candidates are suppressed. This is fast, gives clean, well-separated instances, and is excellent for the roughly convex, blob-like nuclei common in fluorescence imaging; the object-probability map is comparatively noise-tolerant. Its hard geometric prior is also its failure mode: radial rays from a center cannot represent concavities, so non-convex or elongated organelles are systematically clipped or split, and it assumes each object has a visible, unoccluded center — when an overlapping neighbor occludes that center, the polygon estimate collapses. For branched organelles StarDist is structurally the wrong representation.

**Mask R-CNN** is detection-first: a **Region Proposal Network (RPN)** emits candidate boxes, **RoIAlign** crops and resamples backbone features inside each box, and a **per-ROI mask head** segments the single instance within that box; final instances are the per-ROI masks surviving box-level NMS. Because each ROI is segmented independently, the per-ROI masks **can spatially overlap** — this is the one baseline that natively represents genuine amodal overlap/occlusion, where two instances truly share pixels. But detection is the bottleneck: the RPN struggles to place accurate boxes around small, low-contrast, densely clustered objects buried in shot noise, so faint cells are simply never proposed; RoIAlign's fixed crop resolution limits boundary precision exactly at the touching interface; and box-level NMS can suppress a heavily-overlapping *true* positive whose box IoU with a neighbor is high.

**Where the proposed design differs and why it fits noisy overlapping fluorescence.** *Separation mechanism:* the proposed design relies on neither flow direction (Cellpose), a convex shape prior (StarDist), nor box proposals (Mask R-CNN). Instead, the per-instance normalized distance transform manufactures a value valley at every touching seam even where intensity shows no gap, so two adjacent cells present two distinct distance peaks; seeded watershed grows one basin per peak and cuts them at the boundary ridge (ch2). This directly targets the merge failure that a binary mask would suffer. *Noise robustness:* unlike the baselines, the pipeline is explicitly built around the Section 2a Poisson+Gaussian regime — variance-stabilized input, GroupNorm for small heteroscedastic batches, an attention bottleneck for global context to disambiguate overlaps, a heavily up-weighted boundary loss, and noise-tolerant seeding (distance smoothing + min-size filtering) rather than raw-intensity minima. The distance channel is a smooth, region-integrated quantity, inherently less fragile under noise than Cellpose's per-pixel flow direction or a bare boundary map. *Overlap handling — the honest concession:* like Cellpose and unlike Mask R-CNN, the proposed representation produces a **hard partition** — each pixel is assigned to exactly one instance, so it cannot natively represent genuinely co-located, overlapping (amodal) objects the way per-ROI masks can, and watershed can under-segment when two objects fuse without a distance valley. The explicit boundary channel and non-convex-tolerant basins mitigate this (an advantage over StarDist), but the design does not claim to solve amodal overlap; for that regime Mask R-CNN's per-ROI masks remain the appropriate tool.

**Synthesis.** For the specific regime of **noisy + overlapping fluorescence objects**, the mechanisms trade off along a clear axis. **StarDist** is preferable when objects are small, abundant, and reliably convex (nuclei) and speed matters — but fails on concave/branched organelles and occluded centers. **Cellpose** is preferable for densely packed, non-convex cells in moderate noise where a hard partition is acceptable — but its flow field degrades in the dimmest inter-object valleys. **Mask R-CNN** is the choice only when *genuine amodal overlap* must be represented and objects are large and well-contrasted enough to be detected — it is the weakest on small, faint, clustered cells in heavy noise. The **proposed distance-plus-boundary transformer-bottleneck U-Net with seeded watershed** is preferable in the target regime — heavy Poisson+Gaussian noise, thin extreme-minority seams, non-convex morphology — because it re-encodes identity as smooth, noise-robust geometry and cuts at learned boundaries; it concedes only the amodal-overlap case, where a per-ROI (Mask R-CNN-style) or explicit overlap-channel extension would be required.

## 9. Verification notes

- **Independent-verification status.** The problem framing (Section 1), data characteristics (Section 2), the intermediate representation and target encoding (Section 3), and the transformer-bottleneck architecture (Section 4) rest on independently checked foundations. The loss design (Section 5), postprocessing (Section 6), the model-plus-loss code stack (Section 7), and the baseline comparison (Section 8) were authored to be internally self-consistent with those foundations but did **not** pass independent verification; treat their specific parameter defaults, code, and comparative claims as engineering recommendations to validate empirically, not as verified results.
- **Numerical/behavioral caveats.** (1) The generalized Anscombe transform needs estimated `g` and `σ_read`, which are camera-specific and often unknown; variance stabilization is motivated-but-optional and a blind/data-driven denoiser may substitute. (2) The watershed elevation `-dist_s + ch2` assumes ch2 ∈ [0,1] is scale-comparable to the distance map; if the boundary channel trains weak, add a `boundary_weight` multiplier. (3) Per-instance max-normalization of ch1 can over-inflate tiny objects' peaks relative to large objects; consider a global cap or clip if small-object over-segmentation appears. (4) Loss weights are deliberately boundary/geometry-heavy starting points — tune Tversky β and `w₀`/`w_boundary` on an instance-level metric (AP@IoU / Panoptic Quality), never pixel Dice. (5) The learned positional embedding is tied to the 16×16 token grid; variable input sizes require interpolated or relative position encodings.
- **Scope reminders.** The default deliverable is 2D single-plane; the 3D path (Section 1b) is a documented extension, not an implemented artifact. Genuine amodal overlap is not representable in the 3-channel scheme (Section 6/8); a per-instance overlap channel or a Mask R-CNN-style per-ROI head would be required. Illustrative quantities in Section 2b (~<1% seam pixels, 1–3 px seam width) are qualitative, not measured statistics.
