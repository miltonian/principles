# A Modified U-Net for Instance Segmentation of Overlapping Cells and Organelles in Noisy Fluorescence Microscopy

*A technical design document: architecture, loss functions, postprocessing, runnable code, and a cited comparison to Cellpose, StarDist, and Mask R-CNN.*

---

## 1. Problem framing and definition of the task

The objective of this system is **instance segmentation**: assigning a distinct integer label to *every individual* cell or organelle in an image, such that each object is recoverable as a separate entity. Concretely, the output is a label map in which background is `0` and each object occupies a unique, contiguous set of pixels sharing one label. Critically, this requirement holds **including for objects whose pixels touch or overlap** — two adjacent cells of the same type must emerge as two distinct labels, not one merged blob. The entire approach is judged by exactly this property: whether touching and overlapping objects come out as *separate instances*.

This must be sharply distinguished from **semantic segmentation**, which only classifies each pixel as foreground versus background (or by class). A semantic mask can perfectly mark "this pixel is cell" everywhere and still be useless here, because two cells of the same class that touch collapse into a single connected foreground region — semantic segmentation has no mechanism to tell them apart. Instance segmentation is therefore **strictly harder** than semantic segmentation: it must additionally recover object *identity* and *boundaries between like objects*. It is also not the same as detection: a bounding box or a count does not yield the per-object pixel assignment we require. No semantic foreground/background mask, on its own, satisfies this task. Every downstream design decision — architecture, loss, postprocessing — is subordinate to and evaluated against this instance-separation requirement.

**Operating assumptions.** To fix the setting for downstream design, we adopt the following working constraints; the architecture, loss, and postprocessing must satisfy them.

- **Input dimensionality (provisional):** the input is **2D** — either a single focal plane or a projected image (maximum- or mean-intensity Z-projection). This is a working assumption to bound the problem, *not* a claim that the underlying biology is 2D.
- **Noise regime:** the input is **low-SNR, noisy fluorescence**. Expected degradations include shot/Poisson noise, background autofluorescence, and uneven illumination. These degrade object boundaries and make adjacent objects easy to merge — directly threatening the instance-separation property that defines success.
- **Object arrangement:** objects **touch and overlap**, in crowded and dense configurations. The boundaries between adjacent instances are the central difficulty; the interior of objects is comparatively easy. Any downstream mechanism must concentrate its representational and separating power at these inter-object contact regions.

**How 3D would change things.** The 2D assumption is explicitly provisional. Cell and organelle imaging is frequently and genuinely **3D**: confocal and light-sheet microscopy produce volumetric stacks, and biologically meaningful separation of touching objects often exists only across Z. If the setting moves to volumetric data, both the representation and the code change substantively: 2D convolutions become **3D convolutions** throughout; per-pixel label maps become **per-voxel** maps; star-convex polygons (in polygon-based competitors) generalize to star-convex polyhedra sampled over the unit sphere; 2D watershed and 2D connected-component labeling become **3D watershed** with 3D-connected-component labeling over 6-/26-neighborhoods; **anisotropic voxel spacing** (Z resolution ≠ XY) must be handled explicitly in distance metrics and kernel shaping; and the added dimension forces **patch-based or tiled inference** with stitching concerns at patch boundaries. These are forward-looking implications, not a 3D implementation. The choice between 2D and volumetric input is a first-order framing decision that propagates into operators, shape encodings, postprocessing, and the compute budget — and the instance-separation objective holds regardless of dimensionality.

## 2. Evaluation methodology

Success means one thing: **every touching/overlapping cell or organelle emerges as a *separate* labeled instance**, on 2D single-plane or projected low-SNR images. Metrics fall into two families, and only one of them can measure that property.

### 2.1 Metric taxonomy — what actually measures instance separation

**Family (i) — pixel-overlap / semantic metrics**, computed on the *foreground mask* (all object pixels versus background), ignoring instance identity: Jaccard/IoU, Dice (F1 over pixels), and pixel accuracy. Let `P` and `G` be predicted and ground-truth foreground pixel sets. Then

```
IoU(P,G) = |P ∩ G| / |P ∪ G|,     Dice(P,G) = 2|P ∩ G| / (|P| + |G|).
```

These quantify *where the foreground is*, not *how it is partitioned into objects*. They are **invariant to whether the foreground is split into the correct number of instances**: fuse two objects into one blob, or shatter one object into ten, and if the union of pixels is unchanged, IoU and Dice are unchanged.

**Family (ii) — object-level / instance metrics**, computed *after matching* predicted instances to ground-truth instances: object-level Precision, Recall, **F1**, and **Average Precision (AP)** at a fixed IoU threshold. Only these count correctly-*separated* instances, so **these are the primary measures of this system.** Family (i) is retained only as a subordinate pixel-quality sanity check.

### 2.2 Worked demonstration — high Dice/IoU, wrong instances

Take two adjacent ground-truth cells, **A** and **B**, each exactly **100 pixels**, touching. Foreground = 200 px. Arithmetic is intentionally checkable.

**Merge error — one blob covers both.** Predicted instance = the 200-px union of A and B.
- *Pixel metrics:* P = 200, G = 200, intersection = 200. **IoU = 200/200 = 1.000, Dice = 2·200/(200+200) = 1.000.** Pixel-perfect.
- *Object metrics:* the single blob has IoU 100/200 = **0.5** with A and 0.5 with B; under one-to-one matching it can claim at most one. At τ = 0.5 it matches one (TP = 1), leaving the other GT unmatched (FN = 1), no spare prediction (FP = 0): **P = 1.0, R = 0.5, F1 = 0.667, AP = 1/(1+0+1) = 0.5.** At τ = 0.75 the blob clears nothing: TP = 0, FP = 1, FN = 2, **F1 = 0, AP = 0**.

**Split error — one true cell broken in two.** A is predicted correctly (100 px); B is predicted as two 50-px halves.
- *Pixel metrics:* predicted foreground = 100 + 50 + 50 = 200 px = G. **IoU = 1.000, Dice = 1.000.** Again pixel-perfect.
- *Object metrics:* Pred(A) IoU 1.0 with A (TP); each half has IoU 50/100 = 0.5 with B. At τ = 0.5, B matches one half (TP), the other half is unmatched (FP): TP = 2, FP = 1, FN = 0, **P = 0.667, R = 1.0, F1 = 0.800, AP = 2/(2+1+0) = 0.667.** At τ = 0.75 both halves fail: TP = 1, FP = 2, FN = 1, **F1 = 0.400, AP = 0.250.**

In both failures **pixel IoU/Dice = 1.000**, yet object-level F1 and AP collapse — and collapse *further* as the IoU threshold tightens. This is the whole argument: pixel-overlap metrics are blind to merges and splits, so they cannot demonstrate correct separation of touching objects and must never stand as the headline number.

### 2.3 Primary metrics, defined precisely

**Matching step.** Given labeled GT instances `{gᵢ}` and predicted instances `{pⱼ}`, build the IoU matrix `Mᵢⱼ = IoU(gᵢ, pⱼ)`. Fix an IoU threshold τ. Form a **one-to-one assignment** — each prediction matched to at most one GT and vice versa — via **optimal (Hungarian) assignment maximizing total IoU** (a greedy highest-IoU-first pass is an acceptable, near-equivalent alternative; whichever is used must be stated and held fixed). After assignment, keep only pairs with `Mᵢⱼ ≥ τ`. Then:

- **TP** = matched pairs with IoU ≥ τ; **FP** = predictions with no accepted match; **FN** = GT objects with no accepted match.
- **Precision** = TP/(TP+FP), **Recall** = TP/(TP+FN), **F1** = 2·P·R/(P+R) — all at threshold τ.

**Average Precision.** For label-map instance methods (this system, Cellpose, StarDist) there is no natural per-instance confidence ranking, so the **headline AP is the Cellpose/StarDist convention**

```
AP(τ) = TP / (TP + FP + FN),
```

a single-threshold quality score that penalizes false positives (splits/spurious objects) and false negatives (merges/misses) symmetrically. This convention makes the later benchmark apples-to-apples with published Cellpose/StarDist numbers. Where a model *does* emit per-instance scores (e.g., Mask R-CNN), the **detection-style AP** — area under the precision–recall curve obtained by sweeping the score ranking at fixed τ — may additionally be reported, but it is a *different* quantity and must never be numerically merged with `AP(τ) = TP/(TP+FP+FN)` or with COCO's IoU-averaged mask AP.

**Threshold sweep.** Report AP(τ) for **τ ∈ {0.50, 0.55, …, 0.90}**, give **mAP** as their mean, but keep **τ = 0.50** as the single headline comparison point and **object-level F1@0.50** alongside it. *(3D note: under volumetric input the identical object-level F1/AP apply unchanged, with IoU computed over 3D voxel masks.)*

### 2.4 Why the fixed IoU threshold matters

The threshold sets *how good a boundary must be to count as the same object*. A **lenient τ = 0.50** already catches the gross topological errors — the merge and split of §2.2 lose TP and gain FP/FN even at 0.50. **Stricter τ = 0.75–0.90** additionally punishes *boundary sloppiness*: a prediction that separates two touching cells but places the dividing contour imprecisely still clears 0.50 yet fails 0.90. Because the regime is crowded, touching, low-SNR objects whose inter-object boundary is the central difficulty, the strict-threshold behavior is the discriminating measurement. We always report a strict threshold beside the lenient one.

### 2.5 Secondary / diagnostic metrics (strictly subordinate)

Reported only *after* the primary F1/AP, never as evidence of separation: **explicit merge and split counts** (number of GT objects covered by one prediction; predictions splitting one GT); **Panoptic Quality** decomposition `PQ = SQ × RQ`, where `RQ = TP/(TP + ½FP + ½FN)` is a recognition (separation) term and `SQ` = mean IoU of matched pairs is a boundary-quality term; the **Aggregated Jaccard Index (AJI)**; **per-object boundary F-score** for contour precision; and **pixel Dice / IoU** permitted **only** as a foreground-quality sanity check — provably compatible with completely wrong instance counts, therefore never separation evidence.

### 2.6 Comparison-table column specification and reporting hygiene

When this approach is benchmarked against Cellpose, StarDist, and Mask R-CNN, every model is scored with the **identical matching procedure, assignment rule, and thresholds**, on the same images:

| Model | Obj-F1 @ IoU 0.50 | AP @ 0.50 | AP @ 0.75 (strict) | mAP 0.50–0.90 | Merges | Splits |
|-------|------------------|-----------|--------------------|--------------|--------|--------|

AP is the `TP/(TP+FP+FN)` convention throughout; any model's native score (COCO mask AP, etc.) is **recomputed** under this procedure, not copied. **Aggregate per-image, not per-object:** compute TP/FP/FN within each image, then report the **mean across images with its spread (SD or IQR)** — pooling all objects lets a few dense fields dominate and hides variance that matters under noise. State every choice: the IoU threshold set, the assignment rule (Hungarian versus greedy), and the tie-breaking policy (e.g., break equal-IoU ties by larger predicted area, fixed and documented).

`★ Insight ─────────────────────────────────────`
- The **matching step is the load-bearing choice**, not the overlap measure: §2.2 shows IoU/Dice = 1.000 under both a merge and a split, so separation quality only becomes visible once predictions are matched one-to-one and unmatched objects are scored as FP/FN.
- **One-to-one assignment prevents double-counting**: without it, the merge blob would "match" both A and B and score as two TPs, laundering a merge error into a perfect score.
- `AP(τ) = TP/(TP+FP+FN)` folds precision and recall into one number symmetric to splits (FP) and merges (FN) — which is precisely why it, not pixel Dice, is the headline.
`─────────────────────────────────────────────────`

### 2.7 Metric reference implementation

```python
import numpy as np
from scipy.optimize import linear_sum_assignment

def iou_matrix(gt, pred):
    """IoU between every GT instance and every predicted instance.
    gt, pred: 2D int label maps; 0 = background, positive ints = instance IDs.
    (In 3D these are 3D label volumes; the logic is identical.)"""
    gt_ids = np.unique(gt); gt_ids = gt_ids[gt_ids > 0]
    pr_ids = np.unique(pred); pr_ids = pr_ids[pr_ids > 0]
    M = np.zeros((len(gt_ids), len(pr_ids)), dtype=np.float64)
    for i, g in enumerate(gt_ids):
        gmask = (gt == g); garea = gmask.sum()
        for p in np.unique(pred[gmask]):          # only labels touching this GT
            if p == 0:
                continue
            pmask = (pred == p)
            inter = np.logical_and(gmask, pmask).sum()
            union = garea + pmask.sum() - inter
            j = int(np.where(pr_ids == p)[0][0])
            M[i, j] = inter / union
    return M, gt_ids, pr_ids

def match_and_score(gt, pred, tau=0.5):
    """One-to-one match GT<->pred by IoU, then object-level metrics at tau."""
    M, gt_ids, pr_ids = iou_matrix(gt, pred)
    n_gt, n_pr = len(gt_ids), len(pr_ids)
    if n_gt == 0 or n_pr == 0:
        tp = 0
    else:
        row, col = linear_sum_assignment(-M)      # Hungarian maximizes total IoU
        tp = int(sum(M[r, c] >= tau for r, c in zip(row, col)))
    fp = n_pr - tp   # unmatched predictions (e.g. split halves)
    fn = n_gt - tp   # unmatched GT (e.g. merged-away object)
    p  = tp / (tp + fp) if tp + fp else 0.0
    r  = tp / (tp + fn) if tp + fn else 0.0
    f1 = 2 * p * r / (p + r) if p + r else 0.0
    ap = tp / (tp + fp + fn) if (tp + fp + fn) else 0.0  # Cellpose/StarDist AP(tau)
    return dict(tau=tau, tp=tp, fp=fp, fn=fn, precision=p, recall=r, f1=f1, ap=ap)
```

`linear_sum_assignment` on `-M` gives the **optimal one-to-one** pairing; the `>= tau` filter is applied *after* assignment, so a boundary-sloppy but topologically-correct match still counts at τ = 0.50 yet is rejected at τ = 0.75 — the exact strict-threshold behavior §2.4 relies on. `fp = n_pr - tp` and `fn = n_gt - tp` fall straight out of the one-to-one count: every unmatched prediction is a split/spurious object and every unmatched GT is a merge/miss, so no error can hide.

## 3. Noise-handling strategy

This section owns the noise axis — everything upstream of, and feeding into, the segmentation network.

### 3.1 Mechanism — how noise degrades instance *separation* specifically

Fluorescence noise is **signal-dependent**: photon arrivals are **Poisson (shot) noise** whose variance scales with intensity, plus approximately **Gaussian read noise** from the sensor. The consequence is not diffuse "lower accuracy" — it is a targeted corruption of the **high-frequency intensity gradients at object interfaces**, exactly the signal a boundary- or distance-based instance head depends on. This produces **two opposite failure directions**:

- **Over-segmentation / fragmentation:** shot-noise fluctuations *inside* a bright object create spurious high-frequency gradients that look like edges. A boundary/seed head reads these as internal contours and splits one object into several labels.
- **Under-segmentation / merges:** the faint true edge in the thin, low-photon gap *between* two touching objects has the weakest gradient in the scene. Read noise plus low counts wash it out below the surrounding fluctuation level, so the head fails to place a separating boundary and fuses two instances.

Both directions attack the same quantity — the reliability of inter-object contrast — which is why the noise condition must be handled explicitly rather than left to the backbone. The claim under test is that **explicitly handling this condition (denoising front-end and/or noise-robust augmentation) measurably improves instance separation over feeding raw noisy images to the same network** — made falsifiable in §3.4.

### 3.2 Lever A — self-supervised denoising front-end (blind-spot / Noise2Void)

**Principle.** A blind-spot network predicts a **masked center pixel from its surrounding receptive field only**, with the center excluded from its own prediction. Because the network *cannot see* the pixel it must predict, it cannot learn the identity function; it is forced to learn the **spatial signal structure** (predictable from neighbors) while the **pixel-independent noise (given the signal) is not predictable and is therefore not reconstructed** — i.e., it is removed. Crucially this needs **no clean ground-truth targets**, the correct fit for microscopy, where clean references rarely exist.

**Noise-model assumption and its mitigations.** Blind-spot denoising assumes noise is **conditionally independent across pixels given the signal, and zero-mean**. Pure Poisson+Gaussian satisfies this. When stressed: apply a **variance-stabilizing (Anscombe) transform** to make Poisson+Gaussian roughly homoscedastic before denoising (inverting afterward); or use a **probabilistic variant (PN2V-style)** that models the pixel noise distribution explicitly; or, for **structured/correlated noise** (fixed-pattern stripes, line artifacts), use structured-mask variants that widen the blind spot along the correlation axis so correlated noise is not leaked into the prediction. These are named as mitigations only when the assumption breaks — not applied by default.

**Integration mode (recommended: joint, task-aware).** Two options exist: *(i) a frozen/fine-tuned preprocessor* — pretrain the denoiser separately, then run it as a fixed front-end, simple and modular but optimized for a **perceptual proxy (PSNR/MSE)**, not for separation; *(ii) end-to-end joint training* — the denoiser is a front-end whose **self-supervised blind-spot loss is an auxiliary term added to the segmentation loss**, so gradients from the downstream task shape the denoised representation. We recommend **(ii)**. Optimizing for PSNR rewards smoothing, and smoothing preferentially removes the faint inter-object contrast the instance head needs — the very signal §3.1 identifies as fragile. Joint, task-aware denoising instead keeps whatever the separation objective finds useful. The trade-off is exposed as a single knob `lambda` (§7).

### 3.3 Lever B — noise-robust augmentation

Rather than *reduce* input noise, augmentation *hardens* the network by simulating real acquisition variability so it becomes **invariant to noise level** instead of overfitting one SNR regime. Physically motivated set: **signal-dependent Poisson (shot) noise** injected as a function of local intensity (photon budget), not additive constant noise; **Gaussian read noise** added after the Poisson step; **varied SNR / photon budgets** across a wide range (bright to near-photon-starved); **intensity/gamma jitter and background-fluorescence offset** to mimic uneven illumination and autofluorescence; and standard geometric augments (flips, rotations, elastic). Prefer augmentation-only when **labeled data already spans the SNR range** you expect; **add the front-end** when **SNR is very low and inter-object contrast is faint**. The two are **complementary** — augmentation makes the network robust; the front-end cleans the input — and can be combined.

### 3.4 Evaluation / ablation plan (makes the noise-handling claim falsifiable)

Hold **backbone, instance head, label loss, and postprocessing fixed**; vary only the noise-handling switches. Judge on **instance-level** metrics, never pixel PSNR: AP averaged over IoU thresholds **AP@[0.5:0.95]**; **AJI / Panoptic Quality**; explicit **split and merge counts** (which directly test the two mechanisms in §3.1); **F1** over matched instances. **Core comparison:** the same network on **(a)** raw noisy input versus **(b)** denoising front-end and/or noise-robust augmentation. The claim is supported *only if* (b) improves the instance-separation metrics; otherwise it is refuted for this data. **Ablation:** front-end on/off × augmentation on/off (2×2). **`lambda` / denoiser-strength sweep — the honest failure mode:** sweep `lambda`. Expect a **non-monotone curve** — too little denoising leaves noise-driven fragmentation; **too much smears touching objects together and raises the merge count**, hurting AJI/PQ. Denoising strength is a **tunable trade-off, not a free win**. Report the sweep, not a single point.

### 3.5 Noise-handling PyTorch components

```python
import torch
import torch.nn as nn

def blind_spot_mask(img, mask_ratio=0.005, max_shift=2):
    """Noise2Void masking. Assumes noise is conditionally independent across
    pixels given the signal and zero-mean (Poisson shot + Gaussian read). The
    masked center pixel's noise is unpredictable from neighbors, so a network
    that must predict it from its surround learns signal, not noise.
    img: (B,C,H,W) noisy. Returns (net_input, mask); masked pixels replaced by
    an off-center neighbor (breaks identity shortcut); mask=1 at supervised px."""
    B, C, H, W = img.shape
    mask = (torch.rand(B, 1, H, W, device=img.device) < mask_ratio).float()
    sy = int(torch.randint(-max_shift, max_shift + 1, (1,)))
    sx = int(torch.randint(-max_shift, max_shift + 1, (1,)))
    if sy == 0 and sx == 0:
        sx = 1                                   # never copy the masked pixel itself
    donor = torch.roll(img, shifts=(sy, sx), dims=(2, 3))
    return img * (1 - mask) + donor * mask, mask

def masked_denoise_loss(pred, target_noisy, mask):
    """Self-supervised MSE at masked locations only, against the observed noisy
    value (an unbiased regression target under zero-mean noise)."""
    se = (pred - target_noisy) ** 2 * mask
    return se.sum() / mask.sum().clamp_min(1.0)

class PoissonGaussianNoise:
    """Signal-dependent shot noise + Gaussian read noise at randomized SNR
    (photon budget). Expects img in [0,1], shape (C,H,W); INPUT ONLY."""
    def __init__(self, peak_range=(5.0, 200.0), read_sigma_range=(0.0, 0.05)):
        self.peak_range = peak_range             # max expected photons -> SNR knob
        self.read_sigma_range = read_sigma_range
    def __call__(self, img):
        peak = torch.empty(1).uniform_(*self.peak_range).item()
        noisy = torch.poisson(img.clamp_min(0) * peak) / peak   # variance scales w/ signal
        sigma = torch.empty(1).uniform_(*self.read_sigma_range).item()
        noisy = noisy + torch.randn_like(noisy) * sigma          # additive read noise
        return noisy.clamp(0.0, 1.0)

class NoiseAwarePipeline(nn.Module):
    """Denoising front-end -> segmentation network. Joint, task-aware training:
    the denoiser is shaped by BOTH its blind-spot loss and the downstream
    segmentation gradients, so it preserves inter-object contrast, not PSNR."""
    def __init__(self, denoiser, seg_net):
        super().__init__()
        self.denoiser = denoiser
        self.seg_net = seg_net
    def forward(self, noisy):
        denoised = self.denoiser(noisy)
        return denoised, self.seg_net(denoised)

def total_loss(denoised_pred, noisy_in, mask, seg_out, seg_target,
               seg_loss_fn, lam=0.1):
    """total = seg_loss + lam * denoise_loss. lam raises denoising aggressiveness
    (cleaner input) but risks SMEARING closely-touching objects together and
    HURTING separation. Tune lam on instance metrics (section 3.4), never PSNR."""
    d_loss = masked_denoise_loss(denoised_pred, noisy_in, mask)
    s_loss = seg_loss_fn(seg_out, seg_target)
    return s_loss + lam * d_loss, s_loss, d_loss
```

The denoiser emits a denoised tensor that feeds the downstream instance head; no assumption is made about the head's representation (flows, star-convex polygons, boundary/distance maps, RoI masks). A production `blind_spot_mask` should sample donors from a proper local window and exclude the center; the roll-based donor here is the compact form.

## 4. Architecture and instance representation

This section specifies the network backbone and — the load-bearing choice — the **instance-representation output heads**. Everything satisfies two governing requirements: **(i) separate touching/overlapping objects into distinct labels**, and **(ii) remain robust to low SNR**. The denoising front-end (§3) sits upstream; interfaces to the loss (§5), postprocessing (§6), and comparison (§10) are named in §4.4. Working assumption: 2D input, 512×512 training patches, single-channel denoised fluorescence.

### 4.1 Variant selection and justification

We adopt a **Residual–Attention U-Net with a lightweight transformer bottleneck** ("ResAttn-UNet-T"). We commit to all three candidate mechanisms; each is justified against (i) separation and (ii) noise-robustness, and none is decorative.

**Residual blocks in encoder and decoder — ADOPTED.** Pre-activation residual blocks (GroupNorm → SiLU → Conv → GroupNorm → SiLU → Conv, plus identity) are used at every stage. Microscopy instance data is typically small and the input is noisy; residual connections keep gradients healthy at depth so we can afford enough receptive field to see whole crowded objects without vanishing-gradient degradation on limited data. Plain deep stacks train less stably on noisy inputs, harming the fine boundary features separation depends on.

**Attention gates on skip connections — ADOPTED.** Additive attention gates (Oktay-style) filter each encoder skip using a coarser decoder gating signal before fusion. This is the key noise-aware in-network choice: the encoder's high-resolution skips carry exactly the high-frequency content that noise corrupts most. Passing them unfiltered pipes shot-noise-driven spurious edges straight into the decoder, where the boundary/distance/flow heads are computed — producing fragmentation. Attention gates suppress irrelevant/noisy background activations and concentrate skip signal on foreground and on the thin contact regions between touching cells, which is the exact locus of the separation decision.

**Transformer / self-attention bottleneck — ADOPTED (lightweight).** A 2-layer transformer encoder operates on the 32×32 bottleneck. Crowded fields require reasoning about long-range extent — whether two blobs 40 px apart belong to one elongated cell or two touching cells is a global question a purely local receptive field answers poorly. We keep it deliberately small (2 layers) to avoid overfitting on limited noisy data. **Fallback:** if compute is constrained, replace the transformer with a stack of dilated residual convolutions (dilations 1, 2, 4, 8) at the bottleneck — this preserves the global-context intent at lower cost.

### 4.2 Full architecture specification

| Choice | Value | Justification (i = separation, ii = noise) |
|---|---|---|
| Normalization | **GroupNorm, 8 groups** | Small, noisy batches make BatchNorm's batch statistics unreliable and noise-coupled; GroupNorm is batch-size-independent (ii). |
| Activation | **SiLU** | Smooth, non-saturating; stable with residual+GroupNorm blocks. |
| Downsampling | **Strided 3×3 conv (stride 2)** | Learnable downsampling retains more signal than max-pool under noise (max-pool latches onto noise spikes) (ii). |
| Upsampling | **Bilinear ×2 + 3×3 conv** | Avoids transposed-conv checkerboard artifacts that would inject false boundaries (i). |
| Skip fusion | **Attention-gated concat** | See §4.1 (ii). |
| Depth | **4 downsampling stages + bottleneck** | Bottleneck receptive field spans whole crowded objects at 512² input. |

### 4.3 Instance-representation output heads — the centerpiece

**Representation family.** We adopt **dense per-pixel geometric regression**: instead of a single mask, the network emits several overlapping geometric encodings from which downstream postprocessing *reconstructs* individual instances. All heads are predicted at full 512×512 from the shared 32-channel feature map, via a small 2-conv head each:

| Head (tensor name) | Ch | Activation | Geometric quantity | How it enables splitting touching objects |
|---|---|---|---|---|
| `fg_logits` | 1 | sigmoid (logits supervised) | P(foreground) | Restricts all decoding to object pixels; the base mask. Alone it **cannot** separate touching cells. |
| `dist_map` | 1 | sigmoid → [0,1] | normalized distance transform (1 at interior, →0 at own boundary) | Interior maxima are watershed seeds: two touching cells each retain a distinct peak even when foreground is one blob. |
| `boundary_logits` | 1 | sigmoid (logits supervised) | inter-object contact membrane | An explicit carving map: thresholding boundaries breaks the thin bridge between adjacent instances. |
| `flow_field` | 2 | tanh → [-1,1] | unit vector toward each pixel's own instance center | Flow-following clustering advects pixels to per-instance sinks, assigning elongated tails correctly. |

Total output = **5 channels**. The heads are **redundant by design**: distance peaks, flow sinks, and boundary carving are three partially independent cues for the *same* separation. Under low SNR any one can be locally wrong; overlapping encodings let the decoder cross-check them, which is itself a noise-robustness mechanism (ii).

**Why this representation (response to baseline failure modes, §10).**
- **vs. StarDist (star-convex polygons):** star-convex radial-distance shapes cannot represent holes/deep concavities and truncate elongated / non-star-convex cells. Our per-pixel encoding places no shape prior — foreground + boundary can carve holes and arbitrary topology, and the flow field follows a cell's medial structure.
- **vs. Cellpose (flow to center + `diameter`):** Cellpose's own docs state it over-splits when `diameter` is set too small and over-merges when too large. We keep a Cellpose-style flow head for its basin behavior but (a) **normalize the distance head per-instance** so no single global size rescale governs decoding, and (b) rely on a **multi-scale encoder + transformer bottleneck** for scale context rather than a hand-set diameter — removing the single load-bearing hyperparameter.
- **vs. Mask R-CNN (box-proposal masks):** its masks are computed inside axis-aligned RoI proposals — a real representational property; heavily overlapping/adjacent instances sharing a box region are a concern in dense-cell settings. *(Caveat per §10: the original paper reports no overlap artifacts, so we frame this as a property of proposal-bounding, not a quoted failure benchmark.)* Our method is **proposal-free and dense**, so there is no box/NMS bottleneck to cap crowded touching objects.

`★ Insight ─────────────────────────────────────`
- `fg_logits` on its own is a **semantic** mask and is invalid as a sole output: two touching cells collapse into one connected component. The other three heads re-introduce the instance identity a mask throws away.
- Distance-peak seeding and flow-sink clustering fail in *different* ways — distance peaks blur for elongated cells; flow sinks blur for round crowded cells — so predicting both is a hedge, not duplication.
`─────────────────────────────────────────────────`

### 4.4 Interfaces to sibling sections

**Input contract from the §3 denoising front-end.** This network is the `seg_net` in §3's `NoiseAwarePipeline`. It receives the denoised tensor `(B, C_in, H, W)`, `C_in = 1` by default. Under joint training, gradients from all heads flow back into the denoiser.

**For the loss (§5):** define one term (or sub-pair) per named output: `fg_logits` (foreground classification), `dist_map` (normalized-distance regression), `boundary_logits` (touching-interface classification, expect class imbalance), `flow_field` (vector regression). Plus §3's `denoised` auxiliary self-supervised term, combined as `total = seg_loss + λ·denoise_loss`.

**For postprocessing (§6):** consumes `fg_logits` → foreground mask; `dist_map` → seed markers at interior peaks; `flow_field` → per-pixel basin assignment; `boundary_logits` → carving. Intended decoding: threshold foreground, seed a marker-controlled watershed on `dist_map` peaks (reconciled with `flow_field` convergence sinks), and use `boundary_logits` as an elevation/carving term so touching cells split at their contact membrane.

**3D adaptation.** Conv2d→Conv3d (and GroupNorm3d) throughout; every tensor gains a depth axis; downsampling/upsampling become 3D with anisotropic strides if Z ≠ XY; `dist_map` becomes a 3D distance transform, `boundary_logits` a 3D contact surface, and `flow_field` gains a z-component (2→3 channels). The transformer becomes prohibitive over 3D tokens — switch to the dilated-conv fallback or axial attention, and expect patch/tiled inference. Star-convex analogues are irrelevant here since our representation is non-parametric.

## 5. Loss function — formal definitions

The loss supervises exactly the four heads of §4 — **one term (or compound sub-pair) per head; no head is left without a training signal and no term is defined for a head that does not exist.** It is a **compound, region-based loss by construction**: a single unweighted per-pixel loss provably under-learns the thin inter-cell gaps — a tiny pixel fraction — and thus fails to split touching objects. The scalar returned is the `seg_loss_fn` consumed by §3's joint objective `L_joint = L_total + λ_dn·L_denoise`.

### 5.1 Total loss and weighting

```
L_total = λ_Dice·L_Dice + λ_Foc·L_Focal            (foreground head, fg_logits)
        + λ_FT·L_FT + λ_wCE·L_wCE                    (boundary head, boundary_logits)
        + λ_dist·L_dist                              (dist_map)
        + λ_flow·L_flow                              (flow_field)                (1)
```

| Component | Head | Family | Default λ |
|---|---|---|---|
| `L_Dice` | `fg_logits` | region-based (Dice) | 1.0 |
| `L_Focal` | `fg_logits` | imbalance-aware pixel-wise (Focal) | 1.0 |
| `L_FT` | `boundary_logits` | region-based (Focal–Tversky, β>α) | 1.0 |
| `L_wCE` | `boundary_logits` | spatially-weighted cross-entropy, map w(x) | 1.0 |
| `L_dist` | `dist_map` | boundary-weighted regression (MSE) | 1.0 |
| `L_flow` | `flow_field` | masked vector regression (MSE + cosine) | 0.5 |

**All λ are tuned against the object-level metrics of §2 (F1@0.5, AP@0.75, merge/split counts), never against pixel Dice**, since pixel Dice is provably blind to merges and splits. **Notation.** Pixels indexed `i ∈ {1,…,N}`. For a binary head, `zᵢ` is the predicted logit, `p̂ᵢ = σ(zᵢ)` the probability, `yᵢ ∈ {0,1}` the target. `mᵢ = y_iᶠᵍ ∈ {0,1}` is the foreground mask restricting regression heads; `ε = 10⁻⁶`. Binary heads use **sigmoid** (per-channel, independent), *not* softmax, because foreground and boundary are independent binary events.

### 5.2 Foreground head — Dice + Focal

```
L_Dice = 1 - (2·Σ p̂ᵢ y_iᶠᵍ + ε) / (Σ p̂ᵢ + Σ y_iᶠᵍ + ε)                            (2)
```

Dice is a ratio of intersection to summed areas, so it is **insensitive to the foreground/background pixel ratio**: a network cannot minimize it by trivially predicting "background everywhere," the failure mode of an unweighted per-pixel loss on this imbalanced data.

```
L_Focal = -(1/N) Σ αᵢ (1 - p_iᵗ)^γ log p_iᵗ,
  p_iᵗ = p̂ᵢ if y_iᶠᵍ=1 else 1-p̂ᵢ,   αᵢ = α if y_iᶠᵍ=1 else 1-α                     (3)
```

with `γ = 2`, `α = 0.75`. The modulating factor `(1 - p_iᵗ)^γ` **down-weights easy, well-classified majority pixels** so the residual gradient is dominated by the hard minority pixels — the faint, ambiguous foreground borders under low SNR. Dice fixes the region ratio; Focal fixes the per-pixel gradient allocation.

### 5.3 Boundary head — Focal–Tversky + spatially-weighted CE (the imbalance core)

This head carves touching objects apart, and its target — the thin touching-interface membrane — is the *rarest* class in the image. With `b̂ᵢ = σ(z_iᵇ)`, define the soft Tversky index

```
T = (Σ b̂ᵢ y_iᵇ + ε) / (Σ b̂ᵢ y_iᵇ + α_T·Σ b̂ᵢ(1-y_iᵇ) + β_T·Σ(1-b̂ᵢ)y_iᵇ + ε)       (4)
L_FT = (1 - T)^(1/γ_T)                                                             (5)
```

with `α_T = 0.3`, `β_T = 0.7`, `γ_T = 4/3`. Like Dice, `T` is a ratio (insensitive to the huge non-boundary majority). Choosing **`β_T > α_T` makes a false negative on a gap pixel cost more than a false positive**: because true boundary pixels are so scarce, missing them (fusing two cells) is the catastrophic error, so β_T forces recall of those thin gaps. The outer exponent `1/γ_T` (with γ_T > 1) sharpens the loss's slope when `T` is low.

```
L_wCE = -Σ w(xᵢ)[y_iᵇ log b̂ᵢ + (1-y_iᵇ) log(1-b̂ᵢ)] / (Σ w(xᵢ) + ε)                (6)
```

where `w(xᵢ) ≥ 0` is a precomputed per-pixel weight map (Eq. 9) that spikes on the thin gaps between touching instances, in the spirit of the original U-Net separation-border weighting. This is the **explicit spatial up-weighting** lever: it singles out the narrowest gaps — the pixels where two instances are about to merge — multiplying their gradient contribution by an order of magnitude. Focal–Tversky shapes the global boundary *region*; weighted-CE targets individual *hard gap pixels*.

### 5.4 Distance head — boundary-weighted regression

With target `dᵢ ∈ [0,1]` (per-instance normalized distance transform) and `d̂ᵢ ∈ [0,1]`:

```
L_dist = Σ mᵢ vᵢ (d̂ᵢ - dᵢ)² / (Σ mᵢ vᵢ + ε),    vᵢ = 1 + η(1 - dᵢ)                 (7)
```

with `mᵢ = y_iᶠᵍ` restricting to foreground and `η = 4`. A plain MSE over the distance field is dominated by the large, easy interior ridge. The weight `vᵢ = 1 + η(1-dᵢ)` **up-weights the low-distance trough** between touching cells so the network learns a *sharp* trough rather than a smoothed-over one; a blurred trough would let two adjacent basins merge under the watershed.

### 5.5 Flow head — masked vector regression

With target unit vectors `fᵢ = (f_iˣ, f_iʸ)` and prediction `f̂ᵢ ∈ [-1,1]²`, supervised only on foreground:

```
L_flow = Σ mᵢ ‖f̂ᵢ - fᵢ‖² / (2 Σ mᵢ + ε)
       + μ · Σ mᵢ (1 - (f̂ᵢ·fᵢ)/(‖f̂ᵢ‖‖fᵢ‖ + ε)) / (Σ mᵢ + ε)                      (8)
```

with `μ = 0.5`. The mask `mᵢ` keeps background from swamping the signal; the cosine term sharpens vector *direction* precisely where adjacent cells' flows must diverge (at the contact region). Getting the direction right there routes two touching cells' pixels to two different sinks; a magnitude-only MSE would tolerate a near-boundary angular error that fuses the basins.

### 5.6 Boundary weight map (offline target prep)

From the instance label map `L`, compute for every pixel the Euclidean distance `d₁(x)` to the nearest instance and `d₂(x)` to the second-nearest. Then, following the U-Net separation-border weighting,

```
w(x) = w_c(x) + w₀ · exp( -(d₁(x) + d₂(x))² / (2σ²) )                              (9)
```

where `w_c(x)` is a class-balancing weight (inverse boundary frequency), `w₀ = 10`, `σ = 5` px. Pixels in a narrow gap have both `d₁` and `d₂` small, so the exponential is near 1 and `w(x)` spikes; pixels deep inside one object or far in the background revert to `w_c`. This is computed once per image at dataset-build time (needs `scipy.ndimage`) and stored alongside `boundary` targets; the loss `nn.Module` stays pure-PyTorch.

### 5.7 How the loss produces separation

Foreground alone (Dice + Focal) recovers *where* the objects are but, being a semantic mask, collapses two touching cells into one connected component. Separation is produced by the other three heads, each deliberately imbalance-aware: the **boundary head** (`L_FT + L_wCE`) is the primary carving signal — the β_T > α_T asymmetry and the spatial map `w(x)` are *the reason* the network learns the membrane at all; the **distance head** keeps a distinct interior peak per instance with the near-boundary up-weighting `vᵢ` keeping the trough low; the **flow head** makes adjacent cells' vectors diverge at contact. The three cues fail in different regimes, so supervising all three is deliberate redundancy. The common thread: each separating term is region-based or explicitly spatially weighted so the rare separating pixels are not statistically drowned by the majority.

`★ Insight ─────────────────────────────────────`
- Every binary term is computed **from logits** (`logsigmoid`, `binary_cross_entropy_with_logits`), the difference between a stable loss and NaNs on saturated activations — critical because the boundary head, driven toward recall by β_T, pushes logits to extremes.
- The three anti-imbalance mechanisms are **not interchangeable**: Dice/Tversky remove the *ratio* dependence, Focal reallocates the *per-pixel gradient*, and `w(x)` injects *topological* priors the network cannot infer from pixel statistics. Dropping any one re-opens a distinct path to merges/splits.
`─────────────────────────────────────────────────`

## 6. Postprocessing and inference procedure

### 6.1 Inference contract

This stage is purely a decoder: it converts the network's raw output maps into a single-label-per-pixel **int32 instance label map** where 0 = background and 1..N are distinct instances, with touching/overlapping objects carrying different labels. It consumes the four channels: `fg_logits` (→ sigmoid → P(foreground)); `dist_map` (per-instance-normalized, already in [0,1]); `boundary_logits` (→ sigmoid); `flow_field` (2,H,W unit vectors [dy,dx], already tanh-bounded). `fg_logits` alone is a semantic mask that fuses touching cells, so it is never the answer; the other three heads re-inject instance identity.

### 6.2 Step-by-step pipeline

1. **Foreground masking.** `fg = sigmoid(fg_logits) > fg_thresh`, fill small interior holes. Default `fg_thresh = 0.5`; lower to ~0.4 to keep faint low-SNR objects (recall↑, merge-risk↑), raise to ~0.6 when autofluorescence leaks in.
2. **Distance-map conditioning.** `d = gaussian_filter(dist_map, sigma=dist_smooth_sigma)`, masked to fg. Shot noise creates false interior maxima → over-segmentation; smoothing removes sub-object ripples while keeping the one true dome per instance. Default `dist_smooth_sigma = 1.0`.
3. **Seed/marker generation (the crux).** `peak_local_max(d, min_distance=peak_min_distance, threshold_abs=peak_rel_threshold, labels=fg)`, pre-filtered with an h-maxima transform keeping only domes rising ≥ `h_level` above their basin. Because `dist_map` is **per-instance normalized**, two cells fused in one foreground blob still contain two separate interior domes with a saddle between them → one seed per object even in a fused region. Defaults `peak_min_distance = 5` px, `peak_rel_threshold = 0.5`, `h_level = 0.05`.
4. **Flow reconciliation.** Euler-integrate `flow_field` from every fg pixel; peaks of the convergence map are flow sinks (Cellpose-style dynamics). Add any sink far from every distance peak as an extra marker. Distance domes flatten into a ridge for elongated/bent cells so `peak_local_max` may miss/double them; flow integration collapses each elongated object to a single sink. Defaults `flow_steps = 200`, `flow_step_size = 1.0`; toggleable.
5. **Seeded expansion (boundary-carved marker-controlled watershed).** `E = -d + boundary_weight · sigmoid(boundary_logits)`; `labels = watershed(E, markers, mask=fg)`. `-d` seats basins at interior domes and forces the divide onto the distance saddle; adding `sigmoid(boundary_logits)` raises a ridge exactly on the predicted contact membrane so the cut snaps to the learned interface. Every fg pixel is assigned → distinct labels for touching cells. Default `boundary_weight = 1.0`.
6. **Cleanup.** Remove tiny objects (`min_area = 25` px); per-label hole-fill; relabel to contiguous 1..N, cast int32. Because watershed floods the entire fg mask, boundary-membrane pixels are already assigned to the nearest instance — the boundary head only shaped the ridge, it did not delete pixels.

### 6.3 Tunable-parameter table

| Parameter | Default | ↓ over-segmentation | ↓ under-segmentation |
|---|---|---|---|
| `fg_thresh` | 0.5 | raise | lower |
| `dist_smooth_sigma` | 1.0 | raise | lower |
| `h_level` | 0.05 | raise | lower |
| `peak_min_distance` | 5 | raise | lower |
| `peak_rel_threshold` | 0.5 | raise | lower |
| `boundary_weight` | 1.0 | lower→0 | raise |
| `use_flow_reconciliation` | True | disable | enable |
| `min_area` | 25 | raise | lower |

### 6.4 Noise / overlap edge cases

- Spurious peaks from shot noise are suppressed by three stacked mechanisms: Gaussian smoothing (Step 2), the h-maxima dome-depth filter (Step 3), and `peak_local_max`'s `min_distance` peak-merging.
- Faint objects without over-splitting bright ones: `dist_map` is per-instance normalized, so a dim cell's dome still reaches ~1 in its interior and is detected at the same relative `peak_rel_threshold` as a bright cell — seeding is **contrast-invariant by construction** (the Cellpose-`diameter` fix). Retain faint objects via `fg_thresh ≈ 0.4`.
- Elongated cells: a medial ridge gives a flat/duplicated distance dome; Step 4's flow integration collapses the whole cell to one sink.
- **Genuinely OVERLAPPING (co-located) objects — honest limitation:** the output is a single label per pixel, so it cannot assign one pixel to two instances; true transparent superposition is not representable. The `flow_field` sends each shared pixel toward its nearest instance center and the watershed partitions the contested region along the boundary/distance ridge, so heavily overlapping objects are still separated into two labels along the most probable interface — an approximation, not exact overlap recovery.

### 6.5 Inference-robustness notes (optional)

- **Test-time augmentation:** run the 8 dihedral flips/rotations, invert each, and average the maps before decoding — average `sigmoid(fg_logits)`, `dist_map`, `sigmoid(boundary_logits)` directly; for `flow_field` apply the matching vector transform (flip/rotate the [dy,dx] components) before averaging. Never average discrete label maps.
- **Tiled/overlapping inference:** predict on overlapping tiles (e.g. 512² with ≥64 px halo), feather-blend the raw maps in the overlap, then decode **once** on the stitched maps so instances spanning a seam are seeded and watershedded as one object.
- **Instance-level confidence:** score each label from maps already computed — mean `sigmoid(fg_logits)`, `dist_map` peak height, and flow-consistency (fraction of the instance's pixels whose flow points inward). Low-confidence instances are flagged so uncertain separations are surfaced for review rather than silently emitted.

The §3 denoiser strength `λ` couples to these parameters — over-smoothing raises merges, so `boundary_weight`/`peak_min_distance` should be re-tuned jointly on the `λ` sweep.

## 7. Training recipe

Two load-bearing correctness choices: **per-instance normalization of the distance transform** (distances reset at each object → one watershed seed per cell even inside a merged blob), and targets are **recomputed from the augmented label map, never warped** — the `flow_field` target is a vector field that cannot be correctly interpolated by an image warp.

### 7.1 Label / target preparation

```python
import numpy as np
from scipy import ndimage as ndi
from skimage.segmentation import find_boundaries
from typing import Dict

def prepare_targets(instance_mask: np.ndarray, touch_radius: int = 2,
                    touch_weight: float = 10.0, smooth_sigma: float = 1.0
                    ) -> Dict[str, np.ndarray]:
    """Compute all head targets from a 2D integer instance-label map.
    Returns float32 arrays: 'semantic'(H,W){0,1}, 'dist'(H,W)[0,1] PER-INSTANCE,
    'boundary'(H,W){0,1}, 'boundary_weight'(H,W)>=1, 'flow'(2,H,W)[-1,1]."""
    H, W = instance_mask.shape
    ids = np.unique(instance_mask); ids = ids[ids > 0]
    semantic = (instance_mask > 0).astype(np.float32)
    dist = np.zeros((H, W), np.float32)
    flow = np.zeros((2, H, W), np.float32)
    for lbl in ids:
        m = instance_mask == lbl
        edt = ndi.distance_transform_edt(m)          # max at instance center
        peak = edt.max()
        if peak > 0:
            dist[m] = (edt / peak)[m]                 # PER-INSTANCE: resets per object
        edt_s = ndi.gaussian_filter(edt, smooth_sigma)
        gy, gx = np.gradient(edt_s)                   # unit field toward medial center
        flow[0][m] = gy[m]; flow[1][m] = gx[m]
    mag = np.sqrt(flow[0] ** 2 + flow[1] ** 2)
    nz = mag > 1e-8
    flow[0][nz] /= mag[nz]; flow[1][nz] /= mag[nz]
    inner = find_boundaries(instance_mask, mode='inner')
    touch = _touching_interface(instance_mask, touch_radius)
    boundary = (inner | touch).astype(np.float32)
    boundary_weight = np.ones((H, W), np.float32)
    boundary_weight[touch] = touch_weight            # emphasize thin separating gap
    return {"semantic": semantic, "dist": dist, "boundary": boundary,
            "boundary_weight": boundary_weight, "flow": flow.astype(np.float32)}

def _touching_interface(mask: np.ndarray, radius: int) -> np.ndarray:
    """Foreground pixels whose (2r+1) window spans >=2 distinct instance labels."""
    lab = mask.astype(np.int32); size = 2 * radius + 1
    mx = ndi.maximum_filter(lab, size=size)
    lab_no_bg = np.where(lab == 0, mx, lab)          # neutralize bg
    mn = ndi.minimum_filter(lab_no_bg, size=size)
    return (mask > 0) & (mx != mn)
```

Rationale for `touch_weight = 10.0`: touching-interface pixels are typically <1% of foreground, so without up-weighting the boundary head learns to predict "no boundary" everywhere — the exact under-segmentation failure §3.1 warns of.

> **Consistency note (surfaced, not smoothed):** `prepare_targets` emits key `'semantic'`, whereas §5's loss reads `targets['fg']`. Wire them with a one-line alias `targets['fg'] = targets['semantic']` before calling `seg_loss_fn`.

### 7.2 Denoising front-end training & integration

Noise2Void blind-spot masking at `mask_ratio = 0.5%` of pixels per patch (≈1300 supervised pixels at 512²). Masked pixels replaced by an off-center neighbor; masked-MSE loss on masked locations against the observed noisy value. **Joint, task-aware** integration: `L = L_seg + λ·L_denoise`, **λ = 0.1 annealed to 0.05** over the back half. **Warmup:** first **2000 steps** train the denoiser alone so the segmentation net receives a sensible input distribution before its gradients reshape the front-end.

### 7.3 Augmentation pipeline

Ordered, with the application rule explicit: **geometry** transforms both image and label map; **intensity and noise** transform the input only. Targets are **recomputed from the augmented label map** — never warped — because the flow field cannot be correctly interpolated.

| # | Stage | Transform | Params | Applied to |
|---|-------|-----------|--------|-----------|
| 1 | Geometric | H/V flip, 90° rot | p=0.5 each | image + label map |
| 1 | Geometric | Elastic deform | α=40, σ=6, p=0.3 | image + label map |
| 1 | Geometric | Random scale + crop | ±20%, crop 512² | image + label map |
| 2 | **Recompute** | `prepare_targets(aug_labels)` | — | derives dist/boundary/flow |
| 3 | Intensity | Gamma, brightness/contrast | γ∈[0.7,1.5], ±0.2 | **input only** |
| 3 | Intensity | Percentile normalize | 1 / 99.8 clip | **input only** |
| 4 | **Noise** | Poisson shot + Gaussian read | peak∈[5,200], σ_read∈[0,0.05] | **input only** |

Noise last, over the full randomized SNR range: forces invariance to noise level rather than overfitting one regime, and it must follow normalization so photon counts are meaningful.

### 7.4 Optimizer & schedule

```python
from dataclasses import dataclass

@dataclass
class TrainConfig:
    lr: float = 2e-4                # stable AdamW LR for U-Net-scale nets on small sets
    betas: tuple = (0.9, 0.999)
    weight_decay: float = 1e-4      # mild L2; small data -> preserve thin-boundary features
    patch_size: int = 512
    batch_size: int = 4
    grad_accum: int = 2             # -> effective batch 8 (safe under GroupNorm)
    total_steps: int = 100_000
    warmup_steps: int = 1_000       # 1% linear warmup
    lr_min: float = 1e-6            # cosine floor
    denoise_warmup_steps: int = 2_000
    lam_denoise: float = 0.1
    lam_denoise_final: float = 0.05
    mask_ratio: float = 0.005
    grad_clip: float = 1.0
    amp_dtype: str = "bf16"         # avoids fp16 loss-scaling fragility
    ema_decay: float = 0.999        # EMA weights evaluated/checkpointed
    seed: int = 42
    val_every: int = 1_000
    early_stop_patience: int = 15
```

**Rationale.** AdamW over SGD: adaptive steps handle the wildly different gradient scales across the foreground/distance/boundary/flow heads. lr=2e-4 with 1% linear warmup then cosine decay to 1e-6: warmup prevents early AdamW divergence on noisy input. weight_decay=1e-4 (not larger): thin-boundary and faint-flow features are exactly what over-regularization erases. GroupNorm (§4) means no batch-statistic dependence, so effective batch 8 via accumulation is safe. **bf16 AMP** halves memory and needs no GradScaler. **EMA (0.999)** is evaluated/checkpointed for smoother masks and better strict-IoU boundary quality. grad_clip=1.0 because low-SNR batches occasionally spike the loss.

### 7.5 Training loop skeleton

```python
import math, torch

def lr_scale(step, cfg):                          # warmup then cosine
    if step < cfg.warmup_steps:
        return step / max(1, cfg.warmup_steps)
    t = (step - cfg.warmup_steps) / max(1, cfg.total_steps - cfg.warmup_steps)
    cos = 0.5 * (1 + math.cos(math.pi * t))
    return (cfg.lr_min + (cfg.lr - cfg.lr_min) * cos) / cfg.lr

def lam_now(step, cfg):
    return cfg.lam_denoise if step < cfg.total_steps // 2 else cfg.lam_denoise_final

@torch.no_grad()
def ema_update(ema, model, decay):
    for e, p in zip(ema.parameters(), model.parameters()):
        e.mul_(decay).add_(p, alpha=1 - decay)

def train_step(pipeline, ema_pipeline, opt, seg_loss_fn, batch, step, cfg, device):
    """pipeline(net_in) -> (denoised, {fg_logits,dist_map,boundary_logits,flow_field})."""
    x, targets = batch
    x = x.to(device); targets = {k: v.to(device) for k, v in targets.items()}
    targets['fg'] = targets['semantic']                     # alias fix (see 7.1)
    net_in, bs_mask = blind_spot_mask(x, cfg.mask_ratio)    # N2V masking
    with torch.autocast('cuda', dtype=torch.bfloat16):      # bf16, no GradScaler
        denoised, preds = pipeline(net_in)
        l_den = masked_denoise_loss(denoised, x, bs_mask)
        if step < cfg.denoise_warmup_steps:
            loss = l_den                                    # denoiser-only warmup
        else:
            loss = seg_loss_fn(preds, targets)[0] + lam_now(step, cfg) * l_den
    (loss / cfg.grad_accum).backward()
    if (step + 1) % cfg.grad_accum == 0:
        torch.nn.utils.clip_grad_norm_(pipeline.parameters(), cfg.grad_clip)
        for g in opt.param_groups: g['lr'] = cfg.lr * lr_scale(step, cfg)
        opt.step(); opt.zero_grad(set_to_none=True)
        ema_update(ema_pipeline, pipeline, cfg.ema_decay)   # EMA is evaluated/checkpointed
    return float(loss)
```

### 7.6 Validation & stopping

Monitor **object-level AP(τ) = TP/(TP+FP+FN)** (§2) on **EMA weights** decoded through §6 at deployment SNR — headline AP@0.5, strict AP@0.75, mAP@[0.5:0.9]; PQ and merge/split counts logged secondarily; pixel Dice sanity-only. Validate every 1000 steps, early-stop after 15 validations without mAP improvement, checkpoint best-mAP EMA. **Selection is on AP, not loss**, because a lower combined loss can coincide with worse separation if the denoiser over-smooths.

**Recommended noise-vs-clean ablation (makes the noise-handling claim falsifiable).** Holding backbone, loss, and postprocessing fixed, run a **2×2**: denoising front-end {on, off} × noise-robust augmentation {on, off}, evaluated on a held-out **low-SNR** set, plus a **λ sweep** (0 → 0.05 → 0.1 → 0.3). The claim — that explicit noise handling improves instance *separation* (higher AP@0.5/AP@0.75, higher PQ, **lower merge count**) — is supported only if the ablation shows it, with the largest gains expected at strict IoU. **Honest failure mode:** the λ sweep is expected to be non-monotone — too-aggressive denoising smears touching objects together and *raises* merges — so the ablation table must show the sweep.

## 8. Runnable code — model, loss, and separation

### 8.1 Model (`ResAttn-UNet-T`)

Two necessary in-frame engineering decisions: the transformer's learned positional grid is stored small and bilinearly **interpolated** to the runtime token grid (without this, the fixed 256-token bottleneck would crash on a 256×256 test where the bottleneck is 8×8=64 tokens); and `_match_size()` resizes decoder features to skips before gating/concat for odd-size safety.

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


def _match_size(x, ref):
    if x.shape[-2:] != ref.shape[-2:]:
        x = F.interpolate(x, size=ref.shape[-2:], mode='bilinear', align_corners=False)
    return x


class ResBlock(nn.Module):
    """Pre-activation residual: GroupNorm->SiLU->Conv (x2) + identity."""
    def __init__(self, cin, cout, groups=8):
        super().__init__()
        self.n1 = nn.GroupNorm(groups, cin); self.c1 = nn.Conv2d(cin, cout, 3, 1, 1)
        self.n2 = nn.GroupNorm(groups, cout); self.c2 = nn.Conv2d(cout, cout, 3, 1, 1)
        self.skip = nn.Conv2d(cin, cout, 1) if cin != cout else nn.Identity()
    def forward(self, x):
        h = self.c1(F.silu(self.n1(x)))
        h = self.c2(F.silu(self.n2(h)))
        return h + self.skip(x)


class AttentionGate(nn.Module):
    """Oktay-style additive gate: suppresses noisy background skip activations
    so shot-noise edges do not reach the heads."""
    def __init__(self, cx, cg, cint):
        super().__init__()
        self.wx = nn.Conv2d(cx, cint, 1); self.wg = nn.Conv2d(cg, cint, 1)
        self.psi = nn.Conv2d(cint, 1, 1)
    def forward(self, x, g):
        g = F.interpolate(g, size=x.shape[-2:], mode='bilinear', align_corners=False)
        a = torch.sigmoid(self.psi(F.silu(self.wx(x) + self.wg(g))))
        return x * a


class Downsample(nn.Module):
    """Strided 3x3 conv: retains more signal than max-pool under noise."""
    def __init__(self, cin, cout):
        super().__init__(); self.op = nn.Conv2d(cin, cout, 3, 2, 1)
    def forward(self, x):
        return self.op(x)


class Upsample(nn.Module):
    """Bilinear x2 + 3x3 conv: avoids checkerboard artifacts (false boundaries)."""
    def __init__(self, cin, cout):
        super().__init__(); self.conv = nn.Conv2d(cin, cout, 3, 1, 1)
    def forward(self, x):
        x = F.interpolate(x, scale_factor=2, mode='bilinear', align_corners=False)
        return self.conv(x)


class TransformerBottleneck(nn.Module):
    """2-layer pre-LN transformer over the coarse bottleneck grid for global
    crowded-field context. Positional grid interpolated to the runtime token grid."""
    def __init__(self, dim=512, heads=8, layers=2, mlp_ratio=4, pos_grid=16):
        super().__init__()
        self.patch = nn.Conv2d(dim, dim, 2, 2)
        self.unpatch = nn.ConvTranspose2d(dim, dim, 2, 2)
        self.pos = nn.Parameter(torch.zeros(1, dim, pos_grid, pos_grid))
        nn.init.trunc_normal_(self.pos, std=0.02)
        layer = nn.TransformerEncoderLayer(
            d_model=dim, nhead=heads, dim_feedforward=dim * mlp_ratio,
            activation='gelu', batch_first=True, norm_first=True, dropout=0.0)
        self.encoder = nn.TransformerEncoder(layer, num_layers=layers)
    def forward(self, x):
        skip = x
        t = self.patch(x)
        B, C, h, w = t.shape
        t = t + F.interpolate(self.pos, size=(h, w), mode='bilinear', align_corners=False)
        seq = t.flatten(2).transpose(1, 2)
        seq = self.encoder(seq)
        t = seq.transpose(1, 2).reshape(B, C, h, w)
        t = _match_size(self.unpatch(t), skip)
        return skip + t                                    # residual: augment, not replace


class InstanceHeads(nn.Module):
    """Emits the four SEPARABLE heads. A lone fg mask would fuse touching cells."""
    def __init__(self, cf=32, groups=8):
        super().__init__()
        def head(out):
            return nn.Sequential(nn.Conv2d(cf, cf, 3, 1, 1),
                                 nn.GroupNorm(groups, cf), nn.SiLU(),
                                 nn.Conv2d(cf, out, 1))
        self.fg = head(1); self.dist = head(1); self.bnd = head(1); self.flow = head(2)
    def forward(self, f):
        return {'fg_logits': self.fg(f),                   # raw logits
                'dist_map': torch.sigmoid(self.dist(f)),   # [0,1]
                'boundary_logits': self.bnd(f),            # raw logits
                'flow_field': torch.tanh(self.flow(f))}    # [-1,1]


class ResAttnUNetT(nn.Module):
    """Residual-Attention U-Net with transformer bottleneck. Consumes the s3
    denoised tensor (B, in_channels, H, W); returns a dict of 4 heads."""
    def __init__(self, in_channels=1, base=32, groups=8, tf_heads=8, tf_layers=2):
        super().__init__()
        c1, c2, c3, c4, cb = base, base * 2, base * 4, base * 8, base * 16
        self.stem = nn.Conv2d(in_channels, c1, 3, 1, 1)
        self.e1 = nn.Sequential(ResBlock(c1, c1, groups), ResBlock(c1, c1, groups))
        self.d1 = Downsample(c1, c2)
        self.e2 = nn.Sequential(ResBlock(c2, c2, groups), ResBlock(c2, c2, groups))
        self.d2 = Downsample(c2, c3)
        self.e3 = nn.Sequential(ResBlock(c3, c3, groups), ResBlock(c3, c3, groups))
        self.d3 = Downsample(c3, c4)
        self.e4 = nn.Sequential(ResBlock(c4, c4, groups), ResBlock(c4, c4, groups))
        self.d4 = Downsample(c4, cb)
        self.b_pre = ResBlock(cb, cb, groups)
        self.b_tf = TransformerBottleneck(cb, tf_heads, tf_layers)
        self.b_post = ResBlock(cb, cb, groups)
        self.up4 = Upsample(cb, c4); self.ag4 = AttentionGate(c4, c4, c4 // 2)
        self.dec4 = nn.Sequential(ResBlock(c4 * 2, c4, groups), ResBlock(c4, c4, groups))
        self.up3 = Upsample(c4, c3); self.ag3 = AttentionGate(c3, c3, c3 // 2)
        self.dec3 = nn.Sequential(ResBlock(c3 * 2, c3, groups), ResBlock(c3, c3, groups))
        self.up2 = Upsample(c3, c2); self.ag2 = AttentionGate(c2, c2, c2 // 2)
        self.dec2 = nn.Sequential(ResBlock(c2 * 2, c2, groups), ResBlock(c2, c2, groups))
        self.up1 = Upsample(c2, c1); self.ag1 = AttentionGate(c1, c1, c1 // 2)
        self.dec1 = nn.Sequential(ResBlock(c1 * 2, c1, groups), ResBlock(c1, c1, groups))
        self.heads = InstanceHeads(c1, groups)
    def forward(self, x):
        s0 = self.stem(x)
        s1 = self.e1(s0)
        s2 = self.e2(self.d1(s1))
        s3 = self.e3(self.d2(s2))
        s4 = self.e4(self.d3(s3))
        b = self.b_post(self.b_tf(self.b_pre(self.d4(s4))))
        u4 = _match_size(self.up4(b), s4)
        u4 = self.dec4(torch.cat([self.ag4(s4, u4), u4], 1))
        u3 = _match_size(self.up3(u4), s3)
        u3 = self.dec3(torch.cat([self.ag3(s3, u3), u3], 1))
        u2 = _match_size(self.up2(u3), s2)
        u2 = self.dec2(torch.cat([self.ag2(s2, u2), u2], 1))
        u1 = _match_size(self.up1(u2), s1)
        u1 = self.dec1(torch.cat([self.ag1(s1, u1), u1], 1))
        return self.heads(_match_size(u1, x))


if __name__ == '__main__':
    model = ResAttnUNetT(in_channels=1)
    for shape in [(2, 1, 256, 256), (2, 1, 250, 220)]:      # power-of-two + odd
        out = model(torch.randn(*shape))
        print(f'input {shape}:')
        for k, v in out.items():
            print(f'  {k:16s} {tuple(v.shape)}')
        assert out['fg_logits'].shape[1] == 1 and out['dist_map'].shape[1] == 1
        assert out['boundary_logits'].shape[1] == 1 and out['flow_field'].shape[1] == 2
    print('OK: 4 separable heads, odd sizes handled.')
```

### 8.2 Custom compound loss

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class CompoundInstanceLoss(nn.Module):
    """Compound loss for the 4-head instance representation. Returns
    (total_scalar, per_term_dict). Compound/region-based so the rare inter-cell
    gap pixels are not drowned by the majority."""
    def __init__(self, lam_dice=1.0, lam_focal=1.0, lam_ft=1.0, lam_wce=1.0,
                 lam_dist=1.0, lam_flow=0.5, focal_alpha=0.75, focal_gamma=2.0,
                 tversky_alpha=0.3, tversky_beta=0.7, ft_gamma=4.0 / 3.0,
                 dist_eta=4.0, flow_cos_weight=0.5, eps=1e-6):
        super().__init__()
        self.lam_dice, self.lam_focal = lam_dice, lam_focal
        self.lam_ft, self.lam_wce = lam_ft, lam_wce
        self.lam_dist, self.lam_flow = lam_dist, lam_flow
        self.fa, self.fg_ = focal_alpha, focal_gamma
        self.ta, self.tb, self.ft_g = tversky_alpha, tversky_beta, ft_gamma
        self.eta = dist_eta; self.mu = flow_cos_weight; self.eps = eps

    def dice_loss(self, logits, target):                 # Eq.2, ratio => ratio-insensitive
        p = torch.sigmoid(logits).flatten(1); g = target.flatten(1)
        num = 2.0 * (p * g).sum(1) + self.eps
        den = p.sum(1) + g.sum(1) + self.eps
        return (1.0 - num / den).mean()

    def focal_loss(self, logits, target):                # Eq.3, (1-pt)^gamma down-weights easy px
        logp = F.logsigmoid(logits); log1mp = F.logsigmoid(-logits)
        p = torch.sigmoid(logits)
        pt = target * p + (1 - target) * (1 - p)
        logpt = target * logp + (1 - target) * log1mp
        alpha_t = target * self.fa + (1 - target) * (1 - self.fa)
        loss = -alpha_t * (1 - pt).clamp(min=self.eps).pow(self.fg_) * logpt
        return loss.mean()

    def focal_tversky_loss(self, logits, target):        # Eqs.4-5, beta>alpha punishes gap FNs
        p = torch.sigmoid(logits).flatten(1); g = target.flatten(1)
        tp = (p * g).sum(1); fp = (p * (1 - g)).sum(1); fn = ((1 - p) * g).sum(1)
        T = (tp + self.eps) / (tp + self.ta * fp + self.tb * fn + self.eps)
        return (1.0 - T).clamp(min=self.eps).pow(1.0 / self.ft_g).mean()

    def weighted_bce(self, logits, target, w):           # Eq.6, w(x) up-weights thin gaps
        bce = F.binary_cross_entropy_with_logits(logits, target, reduction='none')
        return (w * bce).sum() / (w.sum() + self.eps)

    def distance_loss(self, dist_pred, dist_tgt, fg_mask):  # Eq.7, up-weight basin walls
        v = 1.0 + self.eta * (1.0 - dist_tgt)
        w = fg_mask * v
        se = (dist_pred - dist_tgt) ** 2
        return (w * se).sum() / (w.sum() + self.eps)

    def flow_loss(self, flow_pred, flow_tgt, fg_mask):   # Eq.8, masked MSE + cosine
        m = fg_mask
        se = ((flow_pred - flow_tgt) ** 2).sum(1, keepdim=True)
        mse = (m * se).sum() / (2.0 * m.sum() + self.eps)
        dot = (flow_pred * flow_tgt).sum(1, keepdim=True)
        nrm = (flow_pred.norm(dim=1, keepdim=True) * flow_tgt.norm(dim=1, keepdim=True))
        cos = 1.0 - dot / (nrm + self.eps)
        cos = (m * cos).sum() / (m.sum() + self.eps)
        return mse + self.mu * cos

    def forward(self, predictions, targets):
        fg_logits = predictions['fg_logits']; bnd_logits = predictions['boundary_logits']
        dist_pred = predictions['dist_map']; flow_pred = predictions['flow_field']
        y_fg = targets['fg']; y_bnd = targets['boundary']; y_dist = targets['dist']
        y_flow = targets['flow']; w_map = targets['boundary_weight']; fg_mask = y_fg
        L_dice = self.dice_loss(fg_logits, y_fg)
        L_focal = self.focal_loss(fg_logits, y_fg)
        L_ft = self.focal_tversky_loss(bnd_logits, y_bnd)
        L_wce = self.weighted_bce(bnd_logits, y_bnd, w_map)
        L_dist = self.distance_loss(dist_pred, y_dist, fg_mask)
        L_flow = self.flow_loss(flow_pred, y_flow, fg_mask)
        total = (self.lam_dice * L_dice + self.lam_focal * L_focal
                 + self.lam_ft * L_ft + self.lam_wce * L_wce
                 + self.lam_dist * L_dist + self.lam_flow * L_flow)
        return total, {'total': total.detach(), 'dice': L_dice.detach(),
                       'focal': L_focal.detach(), 'focal_tversky': L_ft.detach(),
                       'weighted_ce': L_wce.detach(), 'distance': L_dist.detach(),
                       'flow': L_flow.detach()}


# ---- Offline target prep (NOT part of the loss module; needs SciPy) ----
def make_boundary_weight_map(label_map, w0=10.0, sigma=5.0):
    """Eq.9: U-Net separation-border weighting. label_map:(H,W) int instance ids.
    Returns w(x): large in thin inter-cell gaps."""
    import numpy as np
    from scipy.ndimage import distance_transform_edt
    ids = [i for i in np.unique(label_map) if i != 0]
    H, W = label_map.shape
    if len(ids) < 2:
        return np.ones((H, W), dtype=np.float32)
    dists = np.stack([distance_transform_edt(label_map != i) for i in ids], 0)
    dists.sort(axis=0)
    d1, d2 = dists[0], dists[1]                          # nearest, second-nearest
    fg = label_map > 0
    wc = np.where(fg, 1.0, fg.mean() / (1.0 - fg.mean() + 1e-6)).astype(np.float32)
    w = wc + w0 * np.exp(-((d1 + d2) ** 2) / (2.0 * sigma ** 2))
    return w.astype(np.float32)
```

### 8.3 Separation / inference (the decisive component)

```python
import numpy as np
from scipy import ndimage as ndi
from skimage.feature import peak_local_max
from skimage.segmentation import watershed, relabel_sequential
from skimage.morphology import (remove_small_objects, remove_small_holes, h_maxima)


def _sigmoid(x):
    return 1.0 / (1.0 + np.exp(-np.asarray(x, dtype=np.float64)))


def _squeeze2d(a):
    a = np.asarray(a)
    return a[0] if a.ndim == 3 and a.shape[0] == 1 else a


def _flow_sinks(flow, fg, steps, step):
    """Euler-integrate the flow field to per-instance convergence sinks.
    flow: (2, H, W) = [dy, dx] in [-1, 1]; returns a convergence-count map."""
    H, W = fg.shape
    fy, fx = flow[0], flow[1]
    ys, xs = np.nonzero(fg)
    py, px = ys.astype(np.float64), xs.astype(np.float64)
    for _ in range(steps):
        iy = np.clip(np.round(py).astype(int), 0, H - 1)
        ix = np.clip(np.round(px).astype(int), 0, W - 1)
        py = np.clip(py + step * fy[iy, ix], 0, H - 1)
        px = np.clip(px + step * fx[iy, ix], 0, W - 1)
    hist = np.zeros((H, W), dtype=np.float64)
    np.add.at(hist, (np.round(py).astype(int), np.round(px).astype(int)), 1.0)
    return ndi.gaussian_filter(hist, 1.0)


def postprocess_instances(fg_logits, dist_map, boundary_logits, flow_field,
                          fg_thresh=0.5, dist_smooth_sigma=1.0,
                          peak_min_distance=5, peak_rel_threshold=0.5,
                          h_level=0.05, boundary_weight=1.0, min_area=25,
                          use_flow_reconciliation=True,
                          flow_steps=200, flow_step_size=1.0):
    """Decode the four output channels into an int32 instance label map.
    Returns labels:(H,W) int32, 0=background, 1..N=instances (touching objects
    receive DISTINCT labels)."""
    fg_logits = _squeeze2d(fg_logits)
    dist_map = _squeeze2d(dist_map)
    boundary_logits = _squeeze2d(boundary_logits)
    flow_field = np.asarray(flow_field)

    # Step 1: foreground mask
    fg = _sigmoid(fg_logits) > fg_thresh
    fg = remove_small_holes(fg, area_threshold=min_area)

    # Step 2: condition distance map (Gaussian smooth, mask to fg)
    d = ndi.gaussian_filter(dist_map.astype(np.float64), dist_smooth_sigma)
    d = d * fg

    # Step 3: distance-peak seeds (one interior dome per instance)
    strong = h_maxima(d, h_level) > 0
    coords = peak_local_max(d, min_distance=peak_min_distance,
                            threshold_abs=peak_rel_threshold,
                            labels=(fg & (strong | (d > peak_rel_threshold))))
    seed_mask = np.zeros(d.shape, dtype=bool)
    seed_mask[tuple(coords.T)] = True

    # Step 4: flow reconciliation (cover elongated cells)
    if use_flow_reconciliation and coords.size:
        sinks = _flow_sinks(flow_field, fg, flow_steps, flow_step_size)
        sink_coords = peak_local_max(sinks, min_distance=peak_min_distance,
                                     threshold_rel=0.1, labels=fg)
        for (y, x) in sink_coords:
            y0, x0 = max(0, y - peak_min_distance), max(0, x - peak_min_distance)
            if not seed_mask[y0:y + peak_min_distance,
                             x0:x + peak_min_distance].any():
                seed_mask[y, x] = True

    markers, n = ndi.label(seed_mask)
    if n == 0:
        return np.zeros(fg.shape, dtype=np.int32)

    # Step 5: boundary-carved marker-controlled watershed
    elevation = -d + boundary_weight * _sigmoid(boundary_logits)
    labels = watershed(elevation, markers=markers, mask=fg)

    # Step 6: cleanup + contiguous relabel
    labels = remove_small_objects(labels, min_size=min_area)
    for lab in np.unique(labels):
        if lab == 0:
            continue
        filled = ndi.binary_fill_holes(labels == lab)
        labels[filled & (labels == 0)] = lab
    labels, _, _ = relabel_sequential(labels)
    return labels.astype(np.int32)
```

Separation is fully traceable: the per-instance-normalized `dist_map` yields one interior dome (hence one seed) per object even inside a fused blob; `sigmoid(boundary_logits)` added to the watershed elevation snaps the cut to the learned contact membrane. For large images, decode once on stitched maps — never merge per-tile labels. The single-label-per-pixel overlap limitation is stated honestly in §6.4.

## 9. Reasoned predicted-contribution analysis

**This is a reasoned, *predicted* contribution analysis, not an executed ablation study** — no model was trained, no benchmark was run, and every "degradation if removed" is a mechanistic prediction (severe / moderate / mild) with its causal reason, never a measured delta. It maps each choice onto two axes: **split errors** (over-segmentation) versus **merge errors** (under-segmentation), and **low-SNR robustness**. The transformer bottleneck *is* present (§4), so it is analyzed as included.

| Component / Loss term | Role | Predicted degradation if REMOVED (severity + mechanism) | Split vs. merge | Low-SNR |
|---|---|---|---|---|
| **Denoising front-end** | cleans input, preserving inter-object contrast | **Severe**: raw noise injects internal gradients and washes out gaps, corrupting head inputs | both worsen | this *is* the low-SNR axis; removal degrades separation most |
| **Residual blocks** | stable gradients at depth | **Mild–moderate**: plain deep stacks train unstably on noise, blurring boundary/distance features | mild merge; can fragment | moderate (worst under noise) |
| **Attention gates** | filter noisy skips | **Moderate**: unfiltered skips pipe shot-noise edges into the heads | more split (fragmentation) | high (skips are most noise-corrupted) |
| **Transformer bottleneck** | global crowded-field context | **Mild–moderate**: local-only answers long-range extent poorly | elongated→split, crowded→merge; small (locally reconstructable) | mild |
| **Instance heads → single fg mask** | encode identity separably | **Catastrophic**: a lone mask is semantic; touching cells are one component | guaranteed merges (anchors the task) | compounded by noise-thinned gaps |
| **Dice term** | ratio-based fg overlap | **Moderate**: drifts to "background everywhere" | missed fg→merge; jitter→split | moderate |
| **Focal term** | down-weight easy pixels | **Mild–moderate**: faint low-SNR borders under-learned | mild merge at contact | high (targets noise-ambiguous pixels) |
| **Focal–Tversky (β>α)** | recall the rare membrane | **Severe**: gaps <1%; without recall the net predicts "no boundary" | merges dominate (primary carve) | high (noise thins the gap) |
| **Weighted-CE w(x)** | up-weight narrowest gaps | **Moderate–severe**: near-merge gaps get negligible gradient | merge of closest pairs | high |
| **Distance regression** | seed peaks | **Moderate–severe**: blurred trough floods basins | weak seeds→merge; noisy→split | moderate |
| **Flow regression** | elongated-cell sinks | **Mild–moderate**: loses single-sink cue | elongated merge/double-count | mild (redundant w/ distance) |

**Cross-component reasoning.** The **denoiser and the boundary/gap terms protect the same fragile low-SNR gap pixels from two directions** — one cleans the input so the gap survives into the features, the other forces loss capacity onto it; removing *both* is qualitatively worse than the sum, and neither fully backs up the other. **Attention gates and the transformer partially overlap** (both add noise-robust context), so removing one is somewhat compensated. The **distance and flow heads mutually compensate** (distance blurs for elongated cells, flow for round crowded cells), so each alone is survivable. But the **instance heads and the watershed postprocessing are jointly non-substitutable**: the heads emit cues no algorithm can invent, and the watershed is the only step that converts them to labels — remove either and separation collapses. Predicted criticality, most to least load-bearing: **(1) instance heads > (2) boundary/gap loss (FT + weighted-CE) > (3) denoising front-end > (4) distance term > (5) attention gates > (6) flow > (7) transformer > (8) Dice > (9) Focal > (10) residual blocks.** *Caveat: all severities and directions above are mechanistic predictions to be confirmed or refuted by the §7 2×2 + λ-sweep ablation; they are not measured results.*

## 10. Comparison to existing models

**This is a reasoned, architectural comparison.** No new head-to-head benchmark was run; every quantitative figure below is quoted from the cited publications and is **not directly comparable across the different datasets and evaluation protocols** those papers used. Any statement about how the proposed network would rank is explicitly marked as *predicted/reasoned*. The primary axis is **not accuracy but instance representation** — *how each method encodes an individual object* — because that determines the *shape* of objects a method can represent at all. Cellpose (Stringer et al., *Nature Methods* 2021), StarDist (Schmidt et al., *MICCAI* 2018; Weigert et al., *WACV* 2020 for 3D), and Mask R-CNN (He et al., *ICCV* 2017) are all strong, widely-used methods; the limitations below are representational trade-offs, not defects. The shared measurement is object-level F1 and `AP(τ) = TP/(TP+FP+FN)` at a fixed IoU (§2).

**Metric-definition warning.** "AP" is not one metric. Cellpose and StarDist define single-threshold `AP(τ) = TP/(TP+FP+FN)`. COCO mask AP (Mask R-CNN) is PR-integrated and averaged over IoU 0.50:0.05:0.95 — a *different quantity*. The `35.7`-style COCO number is never comparable to a `0.6–0.86` cell-dataset AP.

### 10.1 StarDist — star-convex polygons

**Representation.** StarDist encodes each instance as a **star-convex polygon**: from a chosen center it regresses radial distances to the boundary over 32 directions (2D) [https://ar5iv.labs.arxiv.org/html/1806.03535]. Touching objects are separated because each pixel votes for a *whole polygon*, and non-maximum suppression over polygon candidates yields one shape per object.

**Documented shape failure mode.** A star-convex polygon can only represent shapes where every boundary point is visible along a straight ray from the center — a *geometric* limitation, not a tuning deficiency: it (a) **cannot represent objects with holes or deep concavities** (ring/annular or cupped organelles), and (b) **fits strongly elongated or bent cells poorly** (a crescent is not star-convex with respect to any single center, so it is truncated or covered by several polygons — an over-segmentation of one true object). The authors themselves note the "parametric shape model based on only 32 radial directions, instead of a per-pixel segmentation" as the ceiling on high-IoU accuracy [https://ar5iv.labs.arxiv.org/html/1806.03535]. *(The specific "over-segments/truncates elongated cells" phrasing is attributed and unverified against a verbatim source.)*

**Published object-level numbers.** On the DSB2018 nuclei benchmark, using `AP(τ) = TP/(TP+FP+FN)` [https://ar5iv.labs.arxiv.org/html/1806.03535]:

| τ | StarDist | Mask R-CNN | U-Net (3-class) | U-Net (2-class) |
|---|---|---|---|---|
| 0.50 | 0.8641 | 0.8323 | 0.8060 | 0.6739 |
| 0.60 | 0.8043 | 0.7728 | 0.7431 | 0.5975 |
| 0.70 | 0.6850 | 0.6838 | 0.6543 | 0.5339 |
| 0.75 | 0.5862 | 0.5974 | 0.5777 | 0.4819 |
| 0.80 | 0.4495 | 0.4893 | 0.4910 | 0.4151 |
| 0.90 | 0.1191 | 0.1891 | 0.2258 | 0.2032 |

StarDist wins at τ = 0.50 but is *overtaken* at strict thresholds (τ ≥ 0.75) — its coarse parametric boundary caps fidelity where the mask must be tight. Nuclei are near-star-convex, which flatters StarDist; the failure modes bite hardest on the concave/elongated organelles this report targets.

### 10.2 Cellpose — gradient flow fields

**Representation.** Cellpose predicts a per-pixel **XY gradient flow field** pointing toward each object's center, plus a foreground/cell-probability map; instances are recovered by **integrating the flows** so pixels converge into per-object attractor basins [https://cellpose.readthedocs.io/en/v3.1.1.1/settings.html], with the method record at [https://www.nature.com/articles/s41592-020-01018-x] and the code at [https://github.com/MouseLand/cellpose]. This representation is unusually good for arbitrary and elongated shapes, since it places no polygon or box prior on the object.

**Documented sensitivity.** Because flow inference and gradient-tracking dynamics operate at a learned reference scale, Cellpose is **sensitive to the `diameter` (cell-size) parameter**. The authors' own documentation states (verbatim): "When the diameter is set smaller than the true size then cellpose may over-split cells. Similarly, if the diameter is set too big then cellpose may over-merge cells." [https://cellpose.readthedocs.io/en/v3.1.1.1/settings.html]. Images are internally rescaled to a reference size (~30 px cyto / ~17 px nuclei). (Version note: later Cellpose models are reported to be more size-invariant, making `diameter` optional [https://cellpose.readthedocs.io/en/latest/settings.html]; the scale dependence is reduced but is an inherent axis of the flow representation.)

**Published object-level numbers.** On the Cellpose generalist dataset (>70k objects, n=111 test images) at IoU = 0.50, with `AP(τ) = TP/(TP+FP+FN)`: Cellpose ≈ **0.82** versus Mask R-CNN ≈ **0.70** versus StarDist ≈ **0.68**; on a visually-distinct subset 0.73 / 0.55 / 0.54 [https://www.biorxiv.org/content/10.1101/2020.02.02.931238v2.full]. *These specific figures are quoted approximately* — the primary PDF could not be fetched directly (Nature auth, bioRxiv 403, Springer redirect) and the values were extracted from the paper's full text [https://www.biorxiv.org/content/10.1101/2020.02.02.931238v2.full]; read them as "reported strong AP at IoU = 0.5 on the Cellpose generalist dataset," not as exact verified constants.

### 10.3 Mask R-CNN — detect-then-segment, box-bounded masks

**Representation.** Mask R-CNN is a **detect-then-segment** method: a region-proposal network emits axis-aligned bounding boxes, and a small FCN predicts a binary mask **within each RoI box** (an m×m mask resized and thresholded at 0.5) [https://ar5iv.labs.arxiv.org/html/1703.06870]. Instances are separated by having one box (and thus one mask) per object.

**Documented representation property and its consequence.** The representational fact is that **each mask is computed inside, and bounded by, its box proposal**; overlapping proposals are pruned by box-level NMS. From this follows a *reasoned* concern for crowded, non-axis-aligned, or elongated cells: when two such objects' boxes heavily overlap, NMS can suppress one proposal and neighboring masks can be truncated at box edges. **This is a reasoned architectural critique, not a quoted benchmark failure** — indeed, He et al. report that Mask R-CNN "shows no such artifacts" on overlapping objects relative to earlier methods (FCIS) [https://ar5iv.labs.arxiv.org/html/1703.06870]. We therefore present box-bounding as an established property and the dense-crowding limitation as its *predicted* consequence, not as a measured defect.

**Published object-level numbers.** Mask R-CNN's headline figures are **COCO mask AP** — ResNet-101-FPN AP **35.7** / AP50 **58.0** / AP75 **37.8**; ResNeXt-101-FPN 37.1 / 60.0 / 39.4 [https://ar5iv.labs.arxiv.org/html/1703.06870]. COCO mask AP is PR-integrated and averaged over IoU 0.50:0.05:0.95 — a **different quantity** from the single-threshold AP used by Cellpose/StarDist; the two must never be merged. On the *cell* datasets: DSB2018 τ = 0.50 **0.8323** [https://ar5iv.labs.arxiv.org/html/1806.03535] and Cellpose-generalist IoU = 0.50 ≈ 0.70 [https://www.biorxiv.org/content/10.1101/2020.02.02.931238v2.full].

### 10.4 Proposed approach — dense multi-head geometric regression + watershed

**Representation.** The proposed ResAttn-UNet-T emits **dense per-pixel geometric maps rather than a polygon, a single flow, or a box-bounded mask**: `fg_logits`, a **per-instance-normalized** `dist_map`, `boundary_logits`, and a 2-channel `flow_field`. Instances are decoded by seeding a **marker-controlled watershed on distance-map peaks, carving at the predicted boundary membrane, and reconciling seeds with flow sinks** (§6).

**Which limitations this is designed to avoid — by construction.** (i) *vs StarDist:* no shape prior, so foreground + boundary carve holes and arbitrary topology and the flow field follows medial structure — annular and bent/elongated objects are representable. (ii) *vs Cellpose:* keeping a flow head preserves the excellent basin behavior, while **per-instance distance normalization + multi-scale/transformer context** move scale reasoning out of a single hand-set `diameter` — *predicted* to reduce over-split/over-merge scale sensitivity. (iii) *vs Mask R-CNN:* proposal-free and dense, so no box/NMS bottleneck caps crowded touching objects.

**What it trades away — honestly.** The four redundant heads and watershed decode add **compute and several postprocessing hyperparameters** (seed spacing, boundary weight, smoothing), and object scale must be recovered from seed geometry rather than read from a parameter. Being label-map-based, it emits **no natural per-instance confidence ranking**, so detection-style AP is awkward and only `AP(τ) = TP/(TP+FP+FN)` applies cleanly. And like every single-label-per-pixel method, it **cannot represent genuinely co-located (transparent) overlap** — it separates heavily overlapping objects along their most probable interface, an approximation. **No object-level numbers are reported for this method here; its advantages are reasoned from the representation, not measured.**

### 10.6 Why the shared metric is object-level, not pixel-overlap

The table is deliberately built on object-level F1 / AP(τ), not Jaccard/Dice, because **pixel-overlap metrics hide merge and split errors** (§2.2: both failures score pixel IoU/Dice = 1.000 while object F1/AP collapse). Two further cautions from the cited evidence: **"AP" is not one metric** — the 35.7 COCO figure must never be compared to the 0.82/0.86 cell-dataset figures — and the **DSB2018 and Cellpose-generalist tables are different datasets**, so their numbers are not concatenable; no single fetched source evaluates all three on one common dataset+metric. All figures stand *within their own protocol*; any fair head-to-head would require re-scoring every method under the identical matching procedure and thresholds fixed in §2.

**Closing synthesis (reasoned, not empirical).** On the representation axis, the proposed method occupies the **prior-free, dense-encoding** corner. It gives up StarDist's compact parametric shape and Mask R-CNN's clean per-object detection scores, and inherits none of the leaderboard positions those methods have earned. In exchange, it avoids — *by construction of its representation, not by tuning* — StarDist's inability to encode holes and its over-segmentation of elongated cells, Cellpose's single-`diameter` scale sensitivity (via per-instance distance normalization and multi-scale context), and Mask R-CNN's box-bounded, NMS-limited masks in crowded fields. Confirming these predictions requires the object-level F1 / AP(τ) benchmark of §2, run for all methods under one identical protocol. No such measurement is claimed here.

- **Source provenance.** All quantitative figures are citation-gated. Verified by direct fetch: the StarDist DSB2018 AP table and Mask R-CNN COCO AP/AP50/AP75 (ar5iv mirrors of the original papers, [https://ar5iv.labs.arxiv.org/html/1806.03535] and [https://ar5iv.labs.arxiv.org/html/1703.06870]), and the Cellpose diameter over-split/over-merge quote (authors' docs, [https://cellpose.readthedocs.io/en/v3.1.1.1/settings.html], version note [https://cellpose.readthedocs.io/en/latest/settings.html]). Cellpose AP numbers (0.82 / 0.70 / 0.68 at IoU 0.5) are **search-extracted** — Nature ([https://www.nature.com/articles/s41592-020-01018-x]), bioRxiv ([https://www.biorxiv.org/content/10.1101/2020.02.02.931238v2.full]), and Springer all blocked direct fetch — and must be re-verified against the primary PDF before publication; the Cellpose per-threshold (τ > 0.5) table was not located. Code repository: [https://github.com/MouseLand/cellpose].
- **Attributed/unverified claims.** The StarDist "over-segments/truncates elongated cells" phrasing and the Mask R-CNN "box-bounded separation failure" are attributed and unverified; the latter is **contradicted** by He et al.'s own paper ("no such artifacts" on overlapping objects), so it is framed only as a property of proposal-bounding and a reasoned crowding critique, never as a quoted benchmark.
- **No single common benchmark.** No fetched source evaluates all three baselines on one dataset+metric; the DSB2018 (10.1) and Cellpose-generalist (10.2) tables are different datasets and must not be concatenated.
- **Overlap semantics.** The task statement says "touching or overlap," but a single-integer-per-pixel label map cannot represent genuinely *co-located* (transparent, superimposed) objects — only touching/adjacent ones. This document commits to the label-map representation and separates heavily overlapping objects along their most probable interface (§6.4). If the data has true pixel-level overlap, the representation is insufficient: it would require multi-channel/soft instance masks (an added instance/soft-mask head), and the noise front-end would also need soft/multi-mask targets since a denoiser trained to reconstruct signal blends co-located intensities. That is an upstream architecture change, not a postprocessing fix. The IoU-based matching of §2 likewise assumes touching/adjacent, label-map-compatible instances; true pixel overlap would require multi-label IoU over per-instance soft masks.
- **Target-key mismatch (reconciled).** `prepare_targets` (§7.1) emits key `'semantic'`; the §5 loss reads `targets['fg']`. Resolved with a one-line alias in `train_step` (`targets['fg'] = targets['semantic']`); rename at source to remove the seam permanently.
- **Ownership gap in target construction.** The exact `boundary` membrane thickness (px of the touching-interface) and the per-instance flow-target generation are defined in §7.1 (`touch_radius`, gradient-of-EDT flow), but if a separate dataloader owns these, the loss's assumed targets and postprocessing's expectations must be kept consistent or merges will be defined inconsistently.
- **Numerical stability.** `L_flow` divides by predicted/target vector norms with only `+eps` guarding, and `prepare_targets` leaves `flow = 0` on single-pixel/degenerate instances — safe as written because the flow term is foreground-masked; but if any future edit removes the foreground mask on the flow term, background zero-vectors would NaN the cosine. Add an explicit guard if productionized.
- **`dist_map` normalization** is a target-construction property (`prepare_targets`), not enforceable in-model; the network regresses whatever normalized target it is trained on. **`w(x)`** (Eq. 9) requires SciPy and is computed offline; the loss `nn.Module` stays pure-PyTorch.
- **Dataloader compute.** Regenerating distance/boundary/flow targets per augmented sample runs several `distance_transform_edt` calls per crop, which can bottleneck the dataloader for dense fields. Mitigate by budgeting CPU workers, precomputing EDTs and rotating them with correct vector transforms, or using a GPU EDT.
- **Predicted, not measured.** §9's contribution analysis and all "our approach" advantages in §10 are mechanistic predictions to be validated by the §7 2×2 + λ-sweep ablation; no model was trained and no benchmark was run for this document. The `dist_map`/`flow` loss terms are separation-cue supervision, not classic fg/bg class-imbalance terms — their imbalance role is spatial (masking / near-boundary weighting).
